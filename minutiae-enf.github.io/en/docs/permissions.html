Permissions

The node tree is deliberately fine-grained, but you interact with it through two bundles. Read the first section even if you skim the rest; it explains the one behaviour that regularly surprises operators.

How operator defaults work

Every leaf node registers with default: false. A registered false default applies to operators as well - this is standard Bukkit behaviour, not a quirk of this plugin. An operator holds a leaf only through the grant chain:

minutiae.admin (default: op)
  -> minutiae.staff: true
       -> minutiae.command.enforce: true

Consequences worth internalising:

  • If a command vanishes for an operator, the chain is broken - usually by a duplicate mapping key in a hand-edited paper-plugin.yml, which YAML resolves by silently keeping only the last occurrence.
  • Permission plugins that negate minutiae.admin for a group sever everything below it at once, which is also the intended way to build a custom tier.
  • After changing operator status or bundle grants, have the player rejoin: the client command tree is not always re-sent on a live permission change.

The two bundles

minutiae.staff Day-to-day moderation: issuing, lifting (all three kinds, including others' sanctions), dockets, appeal review, diagnostics, and the everyday annotations. default: false; grant it to your moderator group.
minutiae.admin Everything in staff plus the destructive and impersonation surface: bulk lifts, lift reversal, lifting bans, cascades, amendments, aggravating commutation, @as, @backdate, @shadow, reloads. default: op.

With LuckPerms:

/lp group moderator permission set minutiae.staff true
/lp group admin permission set minutiae.admin true

Lift nodes

The lift family splits by consequence rather than by command; the model behind each node is covered in Lifting sanctions.

minutiae.lift.pardon Lift under the pardon kind. In staff.
minutiae.lift.served Lift under the time-served kind. In staff.
minutiae.lift.foreign Lift a sanction issued by someone else. In staff; revoke it for a junior tier that may only undo its own work.
minutiae.lift.blocking Lift a suspension or permanent ban. Admin only by default.
minutiae.lift.cascade Lift a whole joinder chain. Admin only.
minutiae.lift.bulk Bulk-lift by rule or staff criterion. Admin only.
minutiae.lift.reinstate Reverse an erroneous lift. Admin only.

Annotation nodes

Every annotation requires minutiae.annotation.<name> - including annotations supplied by a layout, so a layout cannot grant a moderator a flag their group lacks. The staff bundle carries the everyday set; four are reserved for admin:

minutiae.annotation.shadow Covert isolation is easy to abuse and hard to notice.
minutiae.annotation.as Attribution to another name is impersonation by design.
minutiae.annotation.backdate Rewriting the timeline demands evidence and seniority.
minutiae.annotation.remand Holding actions bypass the normal measure discipline.

Directional and standalone nodes

minutiae.commute.mitigate Commute a layout's measure toward a lighter one. In staff.
minutiae.commute.aggravate Commute toward a harsher one. Admin only - severity increases demand seniority.
minutiae.waive Waive a companion requirement from the relationship matrix. Admin only.
minutiae.amend Modify an existing sanction. Admin only.
minutiae.appeal.submit Submit an appeal. default: true - every player has it.
minutiae.appeal.review Review and decide appeals. In staff.
minutiae.notify.staff Receive staff-channel notifications. In staff.
minutiae.fingerprint.alert Receive evasion alerts. In staff.
minutiae.rank.* Not capability nodes: markers consumed by the rank attribution section of the configuration to label broadcasts. Grant them per group; they gate nothing.

Recipes

A junior helper tier

Staff surface, but only their own non-ban sanctions can be undone, and no covert tooling:

/lp group helper permission set minutiae.staff true
/lp group helper permission set minutiae.lift.foreign false
/lp group helper permission set minutiae.annotation.ghost false

A senior moderator who may lift bans

/lp group srmod permission set minutiae.staff true
/lp group srmod permission set minutiae.lift.blocking true

A read-only auditor

/lp group auditor permission set minutiae.command.info true
/lp group auditor permission set minutiae.command.docket true
/lp group auditor permission set minutiae.command.rules true
Note

Automation (the ALAM engine) and the web panel do not use player permissions: each carries its own bounded grant set in config.yml, so neither can exceed what you explicitly configured even if a token or definition is compromised.
