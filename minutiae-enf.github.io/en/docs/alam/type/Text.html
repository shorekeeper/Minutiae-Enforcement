Text

type

An immutable Unicode string with exact equality and Java regular-expression search through the matching operator.

Syntax

text-type = "Text" ;

text-literal =
    '"',
    { character - '"' - "\"
    | "\" ( '"' | "\" | "n" | "t" | "r" ) },
    '"' ;

Constraints

  • A text literal must end with a closing double quote.
  • Supported escapes are \", \\, \n, \t, and \r.
  • Equality requires another Text value.
  • The ~ operator requires Text on both sides.
  • Text has no ordering, concatenation, indexing, or interpolation operator in the current language.

Semantics

left == rightExact Java string equality.
left != rightNegation of exact equality.
text ~ patternCompiles pattern as a Java regular expression and calls find() on text.

The matching operator searches for a matching region. It does not require the complete text to match. Use anchors such as ^ and $ when full-string matching is required.

Regular expressions follow java.util.regex.Pattern, including Java escaping rules after ALAM string escapes have been decoded.

Under the Java implementation

Escape decoding occurs in the lexer. The decoded string is stored in a small string pool referenced by the token buffer. The parser places the decoded value directly in Expr.TextLit.

Text values are represented by Value.TextV(String). Each use of ~ compiles the pattern when the expression is evaluated. The current implementation does not cache compiled regular expressions.

An invalid pattern produces V015 during compile-time evaluation. During runtime guard evaluation, the same failure is contained and treated as an unsatisfied guard.

Example

rule<Chat> repeated_caps {
    condition {
        trigger = Trigger::Atomic;
        guard = (event) ->
            event.message ~ "^[A-Z ]{20,}$";
    }
    => verdict = warning_verdict<P.2.1>;
}

Diagnostics

L002A text literal is not terminated.
L003A text literal contains an unsupported escape.
E003A text position receives a different type.
E019The matching operator receives a non-text operand.
E032Equality compares text with another type.
V015The right operand of ~ is not a valid Java regular expression.
