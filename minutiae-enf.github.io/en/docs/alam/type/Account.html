Account

type

A nominal identity value representing a subject account independently of its current display name.

Syntax

account-type = "Account" ;

Constraints

  • ALAM has no account literal.
  • An account value normally originates from the subject field of an event.
  • Account is not assignable to Text, even though the runtime representation contains UUID text.
  • Account equality and ordering are not exposed by the current operator set.
  • An unknown subject may be represented at runtime by an empty account value and causes query built-ins to return neutral results.

Semantics

An account denotes the stable UUID identity of an event subject. It does not denote the subject's display name. Use the event field subject_name when human-readable text is required.

Account values may be passed to precedent, fingerprint_score, and is_online, or stored in Sanction.target.

The language intentionally exposes no account construction or parsing function. This prevents source definitions from inventing arbitrary account identities outside event data.

Under the Java implementation

The static type is Type.Prim(PrimKind.ACCOUNT). At runtime, EventAdapter stores the canonical UUID string in Value.TextV while the elaborated field retains type Account.

This representation is an interpreter boundary detail, not an implicit conversion. Built-in functions call UUID.fromString only at the environment boundary. An empty or malformed value becomes Java null.

SanctionMaterializer compares the target UUID text with the completing event subject. If they match, it uses the subject display name on the manual command surface; otherwise it preserves the UUID token.

Example

rule<Login> known_evasion {
    condition {
        trigger = Trigger::Atomic;
        guard = (event) ->
            fingerprint_score(event.subject) >= 0.90;
    }
    => verdict = (event) -> Sanction {
        .target = event.subject,
        .cite = { P.5.1 },
        .measure = Measure::CUSTODY,
        .duration = Duration::Permanent,
        .escalation = Escalation::None,
        .annotations = {},
        .attribution = Attribution::System,
        .dry_run = DryRun::InheritSafety,
    };
}

Diagnostics
