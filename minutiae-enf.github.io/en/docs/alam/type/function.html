Function type

type

A structural type consisting of an ordered parameter list, one result type, and the effect granted to the function body.

Syntax

lambda-expression =
    "(" [ identifier
          { "," identifier } ] ")"
    "->" expression ;

call-expression =
    callee "("
        [ argument { "," argument } ]
    ")" ;

composition-expression =
    outer-function "." inner-function ;

Function types do not have a general source-level type spelling. They are supplied contextually by declarations and fields such as transform, guard, verdict, and Step.guard.

Parameters

ComponentDescription
Parameter types Ordered types accepted by a call. Arity is part of the function type.
Result type Type produced when the function returns successfully.
Effect pure or query, defining the operations allowed in the body.
Parameter names Lexical names declared by a lambda and bound for one application.
Captured environment Compile-time lexical chain retained when the closure is created.

Constraints

  • A lambda requires an expected function type from its surrounding position. ALAM does not infer parameter types from the body.
  • The lambda parameter count must equal the expected function arity.
  • Parameter names must be unique within the lambda.
  • The body must produce the expected result type.
  • A call must supply exactly one argument per parameter.
  • Each argument must be assignable to the corresponding parameter type.
  • The called function's effect must be permitted by the surrounding allowance.
  • Composition requires pure single-parameter functions with compatible intermediate types.

Semantics

Evaluating a lambda creates a closure. It stores the ordered parameter names, body expression, captured environment, and elaborated function type. The body is not evaluated until application.

Applying a closure creates a child environment, binds each argument to the corresponding parameter, and evaluates the body in that child. Bindings are lexical and immutable by language semantics.

A closure observes values available in its captured environment. Under declaration-before-use, a closure cannot capture a later declaration.

Assignability

Function types are contravariant in parameters, covariant in the result, and monotonic in effect:

(TargetParameter -> SourceResult, source-effect)
    assignable to
(SourceParameter -> TargetResult, target-effect)

when:
    TargetParameter is assignable to SourceParameter
    SourceResult is assignable to TargetResult
    source-effect <= target-effect

A pure function can be used where a query-capable function is expected. A query-capable function cannot be placed in a pure position.

Composition

Member syntax denotes composition when its receiver is a pure single-parameter function and the member name resolves to another compatible pure function:

outer.inner(argument)
=
outer(inner(argument))

The same syntax denotes record field access when the receiver is a record or event. Elaboration decides between the two interpretations from the receiver type.

Under the Java implementation

The static representation is Type.Func(List<Type>, Type, Effect). A closure value is Value.ClosureV. Composition is Value.ComposeV. Built-in query functions are represented by Value.BuiltinV.

The compile-time evaluator limits application depth to 512. The runtime interpreter limits it to 256. Exceeding the limit produces V012 at compile time or a contained runtime failure.

Runtime evaluation is reentrant. Each call owns its mutable depth frame. Template-instantiation memoization uses concurrent maps because closures may run on main-thread and asynchronous event dispatch paths.

Effect

pureThe body may compute and construct values but may not read the live enforcement environment.
queryThe body may additionally call precedent, fingerprint_score, is_online, and now.

Example

transform<Int -> Int> double =
    (value) -> value * 2;

template<Rule R>
guard repeat_offender =
    (event) -> precedent(event.subject, R) > 0;

Diagnostics

E003A body or argument has an incompatible type.
E007A query-capable function is called under a pure allowance.
E008A lambda or call has the wrong arity.
E009A lambda appears where no expected function type is available.
E014A lambda parameter name is duplicated.
E016Function composition is invalid or has incompatible intermediate types.
E037A call expression's callee does not have a function type.
V012Compile-time function application exceeds the depth limit.
V013A value expected to be applicable is not a closure, composition, built-in, or accepted guard value.
