Duration

type

A finite non-negative time span represented internally as a signed 64-bit millisecond count.

Syntax

duration-literal =
    digit, { digit }, duration-unit,
    { digit, { digit }, duration-unit } ;

duration-unit = "s" | "m" | "h" | "d" | "w" ;
duration-type = "Duration" ;

Constraints

  • A source literal must alternate non-empty digit runs and unit letters.
  • A finite duration may be zero as a value, but stateful trigger spans and several constructors impose a stricter positive-span constraint.
  • The permanent term is not a value of primitive Duration. It is the nullary constructor Duration::Permanent of the duration-term sum.
  • Duration subtraction and scaling may not produce a negative span.

Semantics

OperationResult
Duration + DurationDuration
Duration - DurationDuration when the result is non-negative
Duration * IntDuration when the factor is non-negative
Int * DurationDuration when the factor is non-negative
< > <= >=Ordering by millisecond magnitude
== !=Exact span equality

Segments are summed. 1h30m denotes 5,400,000 milliseconds. Unit order is not semantically significant, although descending order is the canonical rendering.

Under the Java implementation

The lexer validates the surface alternation. The parser delegates decoding to DurationSpec.parse. The AST stores the parsed value in Expr.DurLit, so later phases do not parse the literal again.

Runtime and compile-time values use Value.DurV(DurationSpec). Scaling uses exact multiplication. Addition through the public duration helper uses exact addition, while expression evaluation separately checks for invalid negative results.

Namespace distinction

The type name Duration denotes a finite primitive span. The path root in Duration::Fixed and Duration::Permanent denotes the separate duration-term sum in the value namespace.

Example

const Duration base = 30m;
const Duration extended = base * 3;

trigger = Trigger::Repeated {
    .count = 5,
    .within = base,
    .group_by = GroupBy::Subject,
};

Diagnostics

L004The surface literal violates digit-unit alternation.
P013The duration decoder rejects the token value.
E019An operator is not defined for the duration operand combination.
V006Duration arithmetic overflows the millisecond representation.
V008Subtraction produces a negative span or scaling uses a negative factor.
