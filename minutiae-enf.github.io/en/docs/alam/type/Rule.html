Rule

type

The rule-identifier primitive and the structural family of rule values parameterized by an ordered event list.

Syntax

rule-identifier-type = "Rule" ;
rule-identifier-literal =
    "P", ( ".", digit, { digit } )+ ;

rule-shape =
    "Rule" "<" event-type
    { "," event-type } ">" ;

Parameters

FormParameterDescription
Rule None Primitive value containing an external rule-registry identifier such as P.2.1.
Rule<Events...> Ordered event types Structural shape of a complete ALAM rule value over those events.

Constraints

  • A rule identifier literal must begin with uppercase P and contain one or more non-empty decimal segments.
  • Every rule identifier reachable from a published layout, guard, verdict, or sequence step guard must exist in the external rule registry.
  • Rule<Events...> requires at least one known event type.
  • A structural rule construction must provide a compatible trigger, guard, and verdict.
  • The primitive Rule and structural Rule<Events...> forms are distinct types and are not assignable to each other.

Semantics

Rule identifiers

A value such as P.2.1 is represented by Value.RuleIdV. It is immutable, supports exact equality, and is admissible as a template parameter. It may be stored in Sanction.cite and Layout.rule or passed to precedent.

Existence in the external registry is checked during verification rather than lexing or elaboration. This keeps the language frontend independent of server configuration while preventing an accepted plan from citing an unknown rule.

Structural rule values

Rule<Chat> or Rule<Login, Chat> describes a residual rule shape. The event list determines the guard and verdict parameter lists and the required trigger shape.

Structural rule values may be generated by a transform and contributed through expand. Explicit rule declarations produce the same residual Value.RuleV form.

Under the Java implementation

The primitive type is Type.Prim(PrimKind.RULE). The structural type is Type.RuleT(List<Type.Event>). Structural equality compares event names position by position.

A residual rule is represented by Value.RuleV, which stores its name, ordered events, reduced trigger variant, guard value, and verdict value.

Example

const Rule flooding_rule = P.2.1;

template<Rule R>
guard has_precedent = (event) ->
    precedent(event.subject, R) > 0;

Diagnostics

R001A rule identifier is absent from the external rule registry.
