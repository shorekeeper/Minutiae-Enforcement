Real

type

An IEEE-754 double-precision floating-point value used for continuous scores and numeric calculations.

Syntax

real-type = "Real" ;
real-literal =
    digit, { digit }, ".", digit, { digit } ;

Constraints

  • A real literal requires at least one digit on both sides of the decimal point.
  • A leading sign is a separate unary operator.
  • ALAM does not implicitly convert Int to Real or Real to Int.
  • Both operands of real arithmetic, ordering, or equality must have type Real.
  • Real cannot scale Duration; duration scaling accepts Int only.

Semantics

+, -, *, /IEEE-754 double arithmetic.
-valueFloating-point negation.
< > <= >=Ordering through Java Double.compare.
== !=Equality through Double.compare, not approximate equality.

Real division by zero follows IEEE-754 behavior and may produce infinity or NaN. It does not produce the integer division diagnostic V005.

The source grammar has no literals for NaN or infinity. Such values may still arise from runtime arithmetic.

Under the Java implementation

The lexer decodes a literal once with Double.parseDouble and stores its raw bits in the token buffer. The AST stores the decoded value in Expr.RealLit. Evaluation represents it as Value.RealV(double).

Runtime event adaptation accepts any Java Number for a real event field and calls doubleValue(). An absent or incompatible fact becomes 0.0.

Example

const Real evasion_threshold = 0.90;

rule<Evasion> high_confidence_evasion {
    condition {
        trigger = Trigger::Atomic;
        guard = (event) -> event.score >= evasion_threshold;
    }
    => verdict = evasion_verdict<Measure::CUSTODY>;
}

Diagnostics

E003A real position receives a different type.
E019An operator is not defined for the supplied operand types.
