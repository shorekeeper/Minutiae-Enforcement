Bool

type

A two-valued logical type whose values are true and false.

Syntax

boolean-type = "Bool" ;
boolean-literal = "true" | "false" ;

boolean-expression =
      "!" boolean-expression
    | boolean-expression "&&" boolean-expression
    | boolean-expression "||" boolean-expression ;

Constraints

  • The operands of !, &&, and || must have type Bool.
  • A rule guard and every template constraint must produce Bool.
  • ALAM does not convert numbers, text, records, or nullary constructors to boolean values.
  • Bool supports equality but not ordering or arithmetic.

Semantics

OperationBehavior
!valueLogical negation.
left && rightEvaluates right only when left is true.
left || rightEvaluates right only when left is false.
==, !=Exact boolean equality and inequality.

Short-circuit behavior applies at compile time and runtime. A query, arithmetic failure, or invalid operation in the unselected right operand is not evaluated.

Under the Java implementation

A boolean value is represented by Value.BoolV(boolean). The compile-time evaluator and runtime interpreter implement conjunction and disjunction before their ordinary two-operand path, so the right expression is evaluated conditionally.

A guard result is accepted only when the interpreter returns Value.BoolV. Any other value or interpreter failure suppresses the firing.

Example

const Bool require_precedent = true;

rule<Chat> repeated_flood {
    condition {
        trigger = Trigger::Atomic;
        guard = (event) ->
            !require_precedent
            || precedent(event.subject, P.2.1) > 0;
    }
    => verdict = flood_verdict<Measure::MUTE, 3h>;
}

Diagnostics

E003A guard, constraint, or other boolean position receives a different type.
E019A boolean operator receives an unsupported operand type.
E032An equality expression compares incompatible types.
