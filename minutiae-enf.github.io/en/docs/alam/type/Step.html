Step

type

An event-parameterized sequence element that pairs one event type with a query-capable admission guard.

Syntax

step-construction =
    "Step" "<" event-type ">" "{"
        ".guard" "=" guard-expression
    "}" ;

Fields

FieldTypeDescription
Event parameter Concrete event type Event kind that may satisfy this sequence position.
guard (EventType) -> Bool Query-capable function deciding whether an incoming event is admitted at this step.

Constraints

  • The event parameter must name a known event type.
  • Exactly one field, guard, is required.
  • The guard must accept one event of the selected type and return Bool.
  • The guard function may have effect pure or query.
  • When used in Trigger::Sequence, the step event must equal the corresponding event declared by the containing rule.

Semantics

A step is a value, not a declaration. It is normally stored in the ordered steps list of Trigger::Sequence.

When an event arrives for a sequence rule, the runtime compares its event kind with the next expected step. If the kinds agree, the step guard is evaluated against that event record. A true result advances the partial match and captures the facts.

A false result does not advance the step. If the same event satisfies the first step while a later step is expected, the sequence may re-anchor to a new partial match beginning at that event.

A step guard failure is treated as a non-match. It does not produce a runtime diagnostic visible to the language source and does not escape into the event handler.

Under the Java implementation

The static type is Type.Step(Type.Event). The evaluated value is Value.StepV, which stores the event type and guard value.

Lowering converts each value to TriggerPlan.Sequence.Step(EventKind, Value). DispatchEngine.stepGuard adapts the incoming facts to the corresponding event record and invokes the guard through the plan's shared Interp.

The base internal step type over Event accepts concrete event steps by the language's deliberate event and step assignability axes.

Effect

query The step guard may read the enforcement environment. Constructing the step value remains pure.

Example

trigger = Trigger::Sequence {
    .within = 10m,
    .group_by = GroupBy::Subject,
    .steps = {
        Step<Login> {
            .guard = Guard::Always,
        },
        Step<Chat> {
            .guard = (event) ->
                event.message ~ "appeal";
        },
    },
};

Diagnostics

E004The construction supplies an unknown field.

E008The guard lambda has a parameter count other than one.
E025The event parameter is unknown.
V002The sequence step event does not correspond to the containing rule event list.
