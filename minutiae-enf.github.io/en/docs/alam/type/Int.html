Int

type

A signed 64-bit integer with checked compile-time and runtime arithmetic.

Syntax

integer-literal = digit, { digit } ;
integer-type = "Int" ;

Constraints

  • A literal must fit the range -9223372036854775808 through 9223372036854775807 after unary negation is applied.
  • ALAM does not implicitly convert Int to Real, Duration, or any other type.
  • Integer match requires a wildcard arm because the finite source cannot enumerate the complete 64-bit domain.

Semantics

+, -, *Checked integer arithmetic.
/Signed integer division with truncation toward zero.
-valueChecked arithmetic negation.
< > <= >=Total signed ordering.
== !=Exact integer equality.
Int * DurationDuration scaling when the integer is non-negative.

The Java evaluator stores an integer as Value.IntV(long). Addition, subtraction, multiplication, and negation use the corresponding Math.*Exact methods. Overflow becomes V006 at compile time or a contained interpreter failure at runtime.

Example

const Int threshold = 5;

trigger = Trigger::Repeated {
    .count = threshold,
    .within = 30s,
    .group_by = GroupBy::Subject,
};

Diagnostics

L005An integer literal exceeds the positive signed 64-bit range during lexing.
E019An operator is not defined for the operand types.
V005Integer division by zero.
V006Checked integer arithmetic overflows.
