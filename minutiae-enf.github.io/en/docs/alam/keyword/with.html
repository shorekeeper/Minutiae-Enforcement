with

keyword

Creates a new value of the same nominal record type by copying a base record and replacing selected fields.

CategoryPostfix value expression
ReceiverNominal Type.Rec only
ResultSame nominal record type as the receiver
MutationNone
Evaluation orderBase first, then replacements in source order
Empty updatePermitted
Java representationFresh Value.RecordV with copied field list

Syntax

base with {
    .field = replacement,
    ...
}
with-expression =
    base-expression
    "with" "{"
        [ field-update
          { "," field-update }
          [ "," ] ]
    "}" ;

field-update =
    "." field-name
    "=" expression ;

Fields

PartRequired typeDescription
BaseNominal recordValue copied before replacements are evaluated.
Field nameExisting member of the base typeSelects one ordered field index.
ReplacementExact field typeValue stored at the selected index in the result.
ResultSame record type as baseFresh immutable record value.

Constraints

Ill-formed if
  1. The base is not a nominal record type.
  2. An update names a field absent from the base record.
  3. A replacement is not assignable to the selected field type.
  4. The same field appears more than once in one update body.
  5. A replacement requires an effect not permitted by the surrounding position.

The receiver must be Type.Rec. The current implementation does not permit with on:

  • concrete event records, which use Type.Event statically;
  • sum variants, including constructors with named fields;
  • Step values;
  • structural Rule<...> values;
  • lists or primitive values.

Semantics

  1. Evaluate the base expression exactly once.
  2. Copy its ordered field-value list.
  3. Evaluate replacement expressions from left to right.
  4. For each replacement, find the field index in the nominal record type.
  5. Replace the value at that index in the copied list.
  6. Publish a new immutable record value.
No sequential update bindings

A replacement expression does not receive an implicit name for the partially updated record. Later replacements cannot refer to earlier replacement results unless those values are independently available through surrounding expressions.

Immutability

The base record and its backing immutable field list remain unchanged. Other constants or closures referencing the base continue to observe the original value.

Empty update

const Layout copy = base_layout with {};

An empty update creates a new Value.RecordV and a copied field list containing equal field references. It does not return the original record object.

Duplicate fields

The parser reports P010 when one field appears more than once. It still retains the duplicate initializers in the temporary AST, but the error prevents successful planning and evaluation publication.

There is therefore no supported last-value-wins behavior for duplicate fields.

Effect

The update operation itself is pure. The base and replacements inherit the effect allowance of the surrounding position.

Implementation notes, non-normative

Elaboration restricts the receiver to Type.Rec:

final Type base =
    infer(update.base(), scope, allowed);

if (!(base instanceof Rec record)) {
    error(
        "E003",
        update.pos(),
        "'with' requires a record value, not "
            + show(base)
    );

    throw Abort.INSTANCE;
}

Each replacement is checked against a linear field lookup:

final int index =
    record.fieldIndex(field.name());

if (index < 0) {
    error(
        "E004",
        field.pos(),
        "record "
            + record.name()
            + " has no field '."
            + field.name()
            + "'"
    );

    throw Abort.INSTANCE;
}

check(
    field.value(),
    record.fields().get(index).type(),
    scope,
    allowed
);

Evaluation clones and updates the field list:

final Value.RecordV base =
    (Value.RecordV) eval(
        update.base(),
        env
    );

final List<Value> fields =
    new ArrayList<>(base.fields());

for (final Expr.FieldInit field
        : update.fields()) {
    fields.set(
        base.type().fieldIndex(
            field.name()
        ),
        eval(field.value(), env)
    );
}

return new Value.RecordV(
    base.type(),
    List.copyOf(fields)
);
Complexity
Base copy
O(F), where F is record field count.
Field lookup
O(F) per replacement.
Total update
O(F + U × F + E), where U is update count and E replacement evaluation cost.
Storage
O(F) references for the new field list.

Example

const Layout base_layout = Layout {
    .key = "chat-base",
    .rule = P.2.1,
    .reason = "Chat violation",
    .measure = Measure::MUTE,
    .duration = Duration::Fixed(1h),
    .escalation = Escalation::None,
    .annotations = {},
};

const Layout severe_layout =
    base_layout with {
        .key = "chat-severe",
        .duration = Duration::Fixed(1d),
        .annotations = {
            Annotation::Notify("staff"),
        },
    };
