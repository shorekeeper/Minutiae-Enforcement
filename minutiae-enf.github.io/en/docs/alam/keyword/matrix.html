matrix

keyword

Declares an ordered homogeneous table whose positional rows are materialized as values of one previously declared record schema.

CategoryTop-level value declaration
IntroducesOne global matrix value name
Declared row typeOne earlier nominal schema
Expression typeSchema[]
Cell effectpure
Row addressingPositional, by schema field order
Compile-time representationValue.ListV of Value.RecordV
Runtime publicationNone unless consumed by later compiled values or expansion

Syntax

matrix<schema-type> name {
    row(cell-list);
    ...
}
matrix-declaration =
    "matrix" "<" schema-type ">"
    identifier "{"
        { row-declaration }
    "}" ;

row-declaration =
    "row" "("
        [ expression
          { "," expression } ]
    ")" ";" ;

Parameters

PartType or formDescription
schema-type Earlier nominal schema Defines row arity, cell types, field names, and positional order.
name Identifier Introduces the complete matrix as a global homogeneous list value.
row Ordered cell list Constructs one record value in schema field order.
Cell expression Corresponding schema field type Pure expression evaluated while the unit is compiled.

Constraints

Ill-formed if
  1. The type argument is unknown at the declaration point.
  2. The type argument does not resolve to a nominal record schema.
  3. The matrix name duplicates any earlier top-level declaration.
  4. A row contains a different number of cells than the schema contains fields.
  5. Cell i is not assignable to schema field i.
  6. A cell expression calls a query-capable function.
  • A matrix may contain no rows.
  • An empty schema may be used by a matrix, and each valid row of that matrix is written as row();.
  • Rows do not name fields and cannot reorder cells independently of the schema.
  • All rows have one exact nominal record type.
  • List element types are invariant; a matrix of one schema is not assignable to a list of another schema with identical fields.
  • Cells may refer only to values and functions visible at the declaration point.

row

A row is a matrix clause, not a general expression and not a separately addressable declaration. Its cells are matched to schema fields by zero-based position.

schema Example {
    Text key;         // field 0
    Rule rule;        // field 1
    Duration span;    // field 2
}

row("chat", P.2.1, 3h);

field 0 <- "chat"
field 1 <- P.2.1
field 2 <- 3h
Positional ABI

The schema's field order is the matrix row ABI. Reordering schema fields changes the interpretation of every row, even when field names remain unchanged.

The matrix syntax provides no named-row alternative. When policy stability matters, treat schema field order as part of the public source contract.

Semantics

Declaration and visibility

The schema must already exist. The matrix symbol is entered into the global symbol table before its cells are checked, following the elaborator's general failure-isolation strategy.

A matrix becomes visible to later accepted declarations. Forward references to a later matrix are not resolved.

Matrix expression type

Referencing the matrix name produces a homogeneous list type:

matrix<TariffRow> tariffs { ... }

tariffs : TariffRow[]

The language does not expose a distinct first-class matrix type. Matrix syntax is a declaration convenience over an ordered list of schema records.

Evaluation order

Rows are evaluated from top to bottom. Within each row, cells are evaluated from left to right.

  1. Create an empty outer row list.
  2. For each row in source order, create an empty cell list.
  3. Evaluate each cell in source order against the global environment.
  4. Create one Value.RecordV using the schema and ordered cell values.
  5. Append that record to the outer list.
  6. After every row succeeds, bind one Value.ListV under the matrix name.

Evaluation of a cell may call earlier pure transforms, instantiate templates in a compatible value context, perform checked arithmetic, construct aggregates, or evaluate match.

Whole-unit publication

If a later cell fails during evaluation, earlier rows may already exist inside the temporary evaluator state. However, the planner publishes no UnitPlan when any evaluation error exists. No partial matrix, layout set, rule set, or automaton from that source unit becomes live.

Immutability

The completed matrix is immutable by language semantics. ALAM exposes no index assignment, insertion, removal, sorting, row mutation, or in-place update.

A row itself is an immutable schema record. with may create a new record value but does not replace the row inside the original matrix.

Available operations

The current expression language exposes no indexing, iteration, filtering, length query, concatenation, or slicing for lists. The principal operation designed for a matrix is expand, which applies one pure transform to every row.

A matrix value may also be passed to a compatible pure function accepting the exact list type, but the current standard library provides no general collection-processing functions.

Expansion order

expand visits rows in matrix source order. That order affects:

  • layout production order;
  • synthesized generated-rule suffixes;
  • which generated layout key is considered the first unit-local producer;
  • the order of generated artifacts retained by the evaluator.
row 0 -> transform-name#0
row 1 -> transform-name#1
row 2 -> transform-name#2

Empty matrix

An empty matrix evaluates to an empty Value.ListV. Expanding it produces no artifacts and no diagnostic.

An empty expansion is not considered a malformed declaration. It may be useful when a generated policy table is intentionally empty in one source version.

Effect

pure Every matrix cell is checked and evaluated under the pure allowance.

A query-capable built-in cannot be used to generate load-time table data. Runtime state belongs in guards, not matrix rows.

Implementation notes, non-normative

The AST preserves row and cell order directly:

record Matrix(
    TypeRef schema,
    String name,
    List<Row> rows,
    Pos pos
) implements Decl {
    record Row(
        List<Expr> cells,
        Pos pos
    ) {
    }
}

Elaboration requires a record schema and checks row arity:

final Type schemaType =
    resolveType(matrix.schema());

if (!(schemaType instanceof Rec rowType)) {
    error(
        "E002",
        matrix.schema().pos(),
        "matrix schema must be a record schema, not "
            + show(schemaType)
    );

    throw Abort.INSTANCE;
}

define(
    matrix.name(),
    new Symbol.MatrixSym(
        matrix.name(),
        rowType,
        matrix,
        matrix.pos()
    ),
    matrix.pos()
);

for (final Decl.Matrix.Row row
        : matrix.rows()) {
    if (row.cells().size()
            != rowType.fields().size()) {
        error(
            "E008",
            row.pos(),
            "row has "
                + row.cells().size()
                + " cell(s); schema '"
                + rowType.name()
                + "' declares "
                + rowType.fields().size()
                + " field(s)"
        );

        continue;
    }

    for (int i = 0;
            i < row.cells().size();
            i++) {
        check(
            row.cells().get(i),
            rowType.fields().get(i).type(),
            emptyScope,
            Effect.PURE
        );
    }
}
Error collection detail

An arity error reports against that row and continues checking later rows. A cell type or effect error uses the elaborator's abort sentinel and abandons the enclosing declaration after reporting.

Evaluation materializes rows as ordinary record values:

final Type.Rec rowType =
    ((Symbol.MatrixSym) requireSymbol(
        matrix.name()
    )).rowType();

final List<Value> items =
    new ArrayList<>(matrix.rows().size());

for (final Decl.Matrix.Row row
        : matrix.rows()) {
    final List<Value> cells =
        new ArrayList<>(row.cells().size());

    for (final Expr cell : row.cells()) {
        cells.add(eval(cell, global));
    }

    items.add(new Value.RecordV(
        rowType,
        List.copyOf(cells)
    ));
}

global.bind(
    matrix.name(),
    new Value.ListV(List.copyOf(items))
);

Name inference exposes the list type:

case Symbol.MatrixSym matrix ->
    new ListT(matrix.rowType());
  • One matrix allocates one outer immutable list and one immutable field list per row.
  • Cell values are not boxed into a separate row wrapper beyond Value.RecordV.
  • The matrix is bound in the evaluator's root environment but is not included in Evaluator.Output.constants.
  • A later closure may capture the root environment containing the matrix, although the language provides few runtime operations over lists.
  • The root environment uses HashMap<String, Value> bindings.
Complexity
Elaboration
O(R × F + T), where R is row count, F schema field count, and T total cell type-checking cost.
Evaluation
O(C + E), where C is total cell count and E total cell-expression cost.
Storage
O(C + R) value references plus one shared schema type.
Expansion
O(R × X), where X is one transform application cost.
Row field access
O(F) name lookup under the current Type.Rec implementation.

Example

schema SanctionPolicy {
    Text key;
    Rule rule;
    Text reason;
    Measure measure;
    Duration duration;
    Int priority;
}

matrix<SanctionPolicy> policies {
    row(
        "chat-flood",
        P.2.1,
        "Repeated message flooding",
        Measure::MUTE,
        3h,
        10
    );

    row(
        "griefing",
        P.3.2,
        "Destruction of player structures",
        Measure::SUSPENSION,
        7d,
        20
    );
}
