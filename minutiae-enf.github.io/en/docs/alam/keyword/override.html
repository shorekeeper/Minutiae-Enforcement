override

keyword

Removes every current automaton member whose principal event matches a selected event type, then inserts one replacement rule.

Syntax

override-item =
    "override" "<" event-type ">"
    "use" replacement-rule ";" ;

Parameters

PartTypeDescription
Event type Concrete event Selects current members for removal by their first declared event.
Replacement rule Earlier rule declaration Rule inserted after the selected members have been removed.

Constraints

  • override is valid only inside an automaton body.
  • The event type must be one of the built-in concrete event types.
  • The replacement must name an earlier rule declaration.
  • The replacement rule's first declared event must equal the selected event type.
  • The replacement rule must not duplicate a member that remains after removal.

Semantics

The item operates on the automaton's current resolved membership, including inherited members and rules inserted by preceding items.

Every member whose first declared event has the selected name is removed. The replacement rule is then inserted at the end of the remaining ordered membership.

For a sequence rule, only the first declared event is its principal event for override purposes. An override for a later step event does not remove that sequence rule.

An override that finds no matching member is legal. The replacement is still inserted, and the compiler emits W001 for operator attention.

Under the Java implementation

Evaluation walks the current LinkedHashMap with its mutable entry iterator. It tests rule.events().get(0).name(), removes every match, and counts removals.

After removal, the evaluator inserts the replacement with put. Elaboration has already established that its principal event matches the selected event.

Because insertion happens after removal, a replacement occupies the order position after every unaffected member, not the position of the first removed rule.

Example

automaton base_watch {
    use chat_length;
    use chat_repetition;
    use login_evasion;
}

automaton strict_chat : base_watch {
    override<Chat> use strict_chat_policy;
}

The derived automaton removes both inherited rules whose principal event is Chat, retains login_evasion, and appends strict_chat_policy.

Diagnostics

P001The event type, use token, replacement name, or semicolon is missing.
E022The replacement does not name a rule or its principal event differs from the selected event.
E025The selected event type is unknown.
V017The replacement duplicates a member that remains in the automaton.
W001No current member was removed for the selected event.
