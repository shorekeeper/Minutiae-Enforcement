override

keyword

Removes every current automaton member whose first declared event equals a selected event type, then appends one replacement rule.

CategoryAutomaton item
Containerautomaton
Selection keyRule's first declared event
Removal scopeAll current members, inherited and local
Replacement positionEnd of remaining membership
No matching memberReplacement still inserted; W001 emitted
Runtime stateNone; resolved at compile time

Syntax

override<event_type> use replacement_rule;
override-item =
    "override" "<" event-type ">"
    "use" replacement-rule ";" ;

Parameters

PartRequired formDescription
event_typeKnown concrete eventSelects current members by principal event.
replacement_ruleEarlier explicit ruleRule appended after matching members are removed.

Constraints

Ill-formed if
  1. The item occurs outside an automaton body.
  2. The event type is unknown.
  3. The replacement does not name an earlier explicit rule.
  4. The replacement rule's first declared event differs from event_type.

The selected event need not currently exist in membership. That condition is a warning rather than an error.

Generated rules cannot be used as replacements because they have no source-addressable explicit rule symbol.

Semantics

Let current be the ordered membership immediately before the item:

  1. Scan every member in order.
  2. Read member.events[0].
  3. Remove the member when that event equals event_type.
  4. Retain all other members in their original relative order.
  5. Append replacement_rule.
current =
    [chat_a, login_a, chat_b, break_a]

override<Chat> use chat_strict;

result =
    [login_a, break_a, chat_strict]

Principal event

The first event in the rule header is the principal event used by override.

rule<Login, Chat> login_then_chat {
    ...
}

This rule is removed by override<Login>, not by override<Chat>.

Current membership

The implementation scans all current members. This includes:

  • rules copied from the parent automaton;
  • rules retained after earlier overrides;
  • rules inserted by earlier use items;
  • replacement rules inserted by earlier overrides.
W001 wording

The current warning text says that the override removed no inherited rule. The implementation actually tests all current membership, not inherited membership only.

Replacement already present

The replacement rule is guaranteed to have the selected principal event. If it is already present in current membership, the removal scan removes it before reinsertion. The item therefore moves it to the end of the remaining membership.

Unlike use, override does not produce V017 for this case.

No matching member

When no member is removed, W001 is emitted and the replacement is still appended. The automaton remains valid and may be published.

Postcondition

After a successful override, the replacement is the only current member with its own rule name. Other rules with the selected principal event have been removed.

Effect

pure Membership replacement is compile-time set algebra and creates no runtime effect itself.

Implementation notes, non-normative

Elaboration validates replacement compatibility:

final Symbol.RuleSym replacement =
    requireRule(
        override.rule(),
        override.pos()
    );

if (!replacement.type()
        .events()
        .get(0)
        .name()
        .equals(override.event())) {
    error(
        "E022",
        override.pos(),
        "rule '" + override.rule()
            + "' is bound to event '"
            + replacement.type()
                .events()
                .get(0)
                .name()
            + "' and cannot replace rules on '"
            + override.event()
            + "'"
    );

    throw Abort.INSTANCE;
}

Evaluation removes through the ordered map iterator:

int removed = 0;
final var iterator =
    current.entrySet().iterator();

while (iterator.hasNext()) {
    if (iterator.next()
            .getValue()
            .events()
            .get(0)
            .name()
            .equals(override.event())) {
        iterator.remove();
        removed++;
    }
}

The warning does not prevent insertion:

if (removed == 0) {
    diags.warning(
        "W001",
        override.pos().line(),
        override.pos().column(),
        "override on '"
            + override.event()
            + "' removed no inherited rule"
    );
}

current.put(
    override.rule(),
    rules.get(override.rule())
);
  • current is a LinkedHashMap.
  • Map.put appends an absent replacement at the end.
  • If an existing replacement survived, put would preserve its current position, but valid event compatibility ensures it was selected and removed first.
  • The resolved immutable automaton list contains no override metadata.
Complexity
Validation
O(1) after symbol lookup.
Removal
O(M), where M is current membership count.
Insertion
Expected O(1).
Additional storage
O(1), excluding the resulting membership list.

Example

automaton base_watch {
    use chat_length;
    use chat_repetition;
    use login_evasion;
}

automaton strict_chat : base_watch {
    override<Chat> use strict_chat_policy;
}
