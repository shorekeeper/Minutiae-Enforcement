const

keyword

Declares one immutable named value whose initializer is evaluated once while the source unit is compiled.

CategoryTop-level value declaration
IntroducesOne global value name
Initializer effectpure
Evaluation timeCompile time
Re-evaluation on accessNone
MutabilityImmutable
Java representationOne Value bound in the root Env

Syntax

const type name = initializer;
const-declaration =
    "const" type identifier
    "=" expression ";" ;

Parameters

PartFormDescription
typeResolvable source typeExact expected type of the initializer.
nameIdentifierGlobal value name visible to later declarations.
initializerPure expression of the declared typeExpression evaluated exactly once during compilation.

Constraints

Ill-formed if
  1. The declared type is unknown at the declaration point.
  2. The name duplicates any earlier top-level declaration.
  3. The initializer is not assignable to the declared type.
  4. The initializer calls a query-capable function.
  5. The initializer references an unknown or later value.
  6. Initializer evaluation fails.

Type availability

The declaration uses the ordinary source type grammar. Common admissible forms include primitive types, Measure, Sanction, Layout, earlier schemas, list types, and structural Rule<Events...> types.

Several built-in constructor roots are not registered as general source type names. For example, a value such as Escalation::None exists, but the current type namespace does not accept Escalation in a const header.

Function constants

The Java type model and composition logic can represent a constant whose type is Type.Func. The current source type grammar has no general function-type spelling, so a source-level const cannot directly declare an arbitrary function type.

Semantics

The initializer is evaluated against the root compile-time environment. Its resulting value is stored under the constant name and returned directly on every later reference.

const Int limit = 2 * 5;

Declaration before use

A constant may reference earlier constants, matrices, transforms, built-in constructors, and pure functions. It cannot reference a later declaration.

The elaborator enters the constant symbol after resolving its header and before checking the initializer. This limits cascading diagnostics, but a self-reference does not become a valid recursive value. The complete unit is rejected when the initializer is invalid.

Aggregate values

A constant may store an immutable list, record, sum variant, layout, sanction descriptor, or structural rule value when the declared source type can express that value.

Referencing an aggregate constant returns the same immutable value object. ALAM provides no mutation operation that can alter the shared value.

Closure capture

A later transform, template body, guard, or verdict may capture a constant through its lexical environment. The constant value is already evaluated before the closure is created.

A runtime guard therefore does not re-run the constant initializer and cannot observe changed external state through it.

Evaluation failure

Checked arithmetic, division by zero, invalid duration arithmetic, invalid text patterns, template constraint failure, and depth exhaustion may fail the initializer.

The evaluator reports the diagnostic and skips binding the failed value. Because the planner publishes no unit after any evaluation error, later temporary evaluator state cannot become live.

Effect

pure The initializer cannot read precedent, fingerprint score, online state, or wall-clock time.

Implementation notes, non-normative

Elaboration resolves and registers the header before checking the initializer:

final Type type =
    resolveType(constant.type());

define(
    constant.name(),
    new Symbol.ConstSym(
        constant.name(),
        type,
        constant.init(),
        constant.pos()
    ),
    constant.pos()
);

check(
    constant.init(),
    type,
    new Scope(null),
    Effect.PURE
);

Evaluation computes and stores one value:

case Decl.Const constant -> {
    final Value value =
        eval(constant.init(), global);

    constants.put(
        constant.name(),
        value
    );

    global.bind(
        constant.name(),
        value
    );
}

Name evaluation returns the existing binding:

case Symbol.Local local ->
    env.lookup(local.name());

case null, default ->
    global.lookup(name.name());
  • The evaluator's constant collection is a declaration-order LinkedHashMap.
  • The public evaluator output wraps it with Map.copyOf; callers should not rely on iteration order of that copied map.
  • The root environment uses a mutable HashMap that grows during evaluation.
  • Values themselves are immutable by language convention.
  • A failed unit is never partially published by Planner.

Example

const Int flood_count = 5;
const Duration flood_window = 30s;
const Text staff_channel = "staff";

const Layout chat_layout = Layout {
    .key = "chat-flood",
    .rule = P.2.1,
    .reason = "Repeated message flooding",
    .measure = Measure::MUTE,
    .duration = Duration::Fixed(3h),
    .escalation = Escalation::None,
    .annotations = {
        Annotation::Notify(staff_channel),
    },
};
