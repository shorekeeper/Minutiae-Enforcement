Sanction

record

A total verdict descriptor containing every value required to resolve an automatic firing onto the ordinary sanction command surface.

Fields

FieldTypeRequiredDescription

Constraints

Structural requirements
  1. Every field is present exactly once.
  2. No unknown field is present.
  3. Every field expression has the exact declared type.
  4. The construction occurs in a position expecting Sanction, normally a verdict body.
  5. The containing verdict remains pure.
Resolution requirements after materialization
  1. At least one cited rule is normally required to establish the primary rule.
  2. Every cited rule identifier must exist in the external rule registry.
  3. The measure and duration must form a coherent command sanction.
  4. Every materialized annotation must be known, structurally valid, and permitted to the configured system actor.
  5. The effective annotation set must satisfy implication, conflict, and companion-requirement rules.
  6. A temporal measure must resolve a finite duration.

The ALAM type checker proves the record shape. It does not by itself prove external rule existence, system-actor permissions, command annotation relationships, configured notification channels, or target resolvability.

Semantics

A Sanction value is returned by a verdict only after the containing trigger and rule guard have passed. The value is not written directly to storage. It is first translated into ParsedCommand, the same syntactic structure produced by the manual /enforce command parser.

target

The target is represented inside the interpreter as canonical UUID text carried under the nominal Account type.

When the target equals the completing event's subject and that event has a non-empty display name, materialization uses the display name as the command target. Otherwise it uses the UUID text.

Completing-event rule

For a sequence verdict, target display-name substitution is based on the event that completed the sequence, even though the target expression may have been read from an earlier captured event argument.

cite

Every cited rule becomes one command directive:

Rule[] { P.2.1, P.2.4 }

->

@count(P.2.1)
@count(P.2.4)

Because an ALAM sanction does not select a layout, the resolver promotes the first count to the primary rule. Remaining entries become additional cited provisions.

An empty citation list leaves the sanction without a primary rule. Some enforcement mechanisms can still resolve, but rule-scoped precedent, warn-first, decay interpretation, and external rule attribution become limited or absent. A total descriptor should normally cite at least one rule.

measure

The constructor name becomes the command measure directive:

Measure::MUTE
->
@measure(MUTE)

The shared resolver applies the same temporal and permission rules as a manual sanction. The measure may later be changed to Measure::WARN by the warn-first execution gate.

duration

Duration::Fixed becomes a canonical duration= override. Duration::Permanent emits no override. The resolver derives the final base duration from the measure's temporal class.

TermMaterialized command data
Duration::Fixed(3h)duration=3h
Duration::PermanentNo duration override

escalation

Escalation::None emits no directive. Escalation::Steps emits @escalate.

Transport limitation

The individual rung values of Escalation::Steps are not transported through ParsedCommand. The direct ALAM sanction has no originating layout, so its resolved escalation array is empty. Construct complete ladders in stamped Layout values when runtime rung selection is required.

annotations

Each annotation constructor is translated to its command equivalent. Most become RawAnnotation tokens. Annotation::Reason becomes the scalar reason= override.

The resolver merges effective annotations by name. Repeated constructors with the same command name therefore use last-value-wins behavior rather than preserving all copies.

attribution

Attribution::Staff becomes @as(name). Attribution::System asks the dispatch engine to add the configured system actor name when the actor has attribution permission.

dry_run

DryRun::Forced marks the materialized sanction as dry-run. DryRun::InheritSafety defers to simulation and automaton arming. No constructor can force live effect.

Resolution and execution

The materialized command is resolved with the system actor's permission predicate. On success, the ordinary sanction executor:

  1. resolves the target account;
  2. loads precedent asynchronously;
  3. evaluates warn-first and duration policy;
  4. calculates expiry and behavioral bits;
  5. returns a preview when dry-run;
  6. otherwise persists the sanction and auxiliary records;
  7. applies player effects on the main thread;
  8. publishes network directives when required;
  9. reports, broadcasts, and notifies according to policy.
Failure behavior

A verdict interpreter failure, materialization inconsistency, or resolver rejection suppresses sanction issuance. No partial sanction is intentionally published by the ALAM dispatch path.

Effect

pure Constructing the descriptor is pure. The later resolver and executor perform queries and effects outside the verdict expression.

Implementation notes, non-normative

The built-in record shape is declared in one ordered list:

public static final Rec SANCTION =
    new Rec("Sanction", List.of(
        new Field("target", ACCOUNT),
        new Field("cite", new ListT(RULE)),
        new Field("measure", MEASURE),
        new Field("duration", DURATION_TERM),
        new Field("escalation", ESCALATION_RESOLVED),
        new Field("annotations", new ListT(ANNOTATION)),
        new Field("attribution", ATTRIBUTION),
        new Field("dry_run", DRY_RUN)
    ));

Record evaluation places source initializers into schema order:

final List<Value> ordered =
    new ArrayList<>(r.type().fields().size());

for (final Type.Field field : r.type().fields()) {
    ordered.add(byName.get(field.name()));
}

yield new Value.RecordV(
    r.type(),
    List.copyOf(ordered)
);

The materializer builds the command bridge:

return new Materialized(
    new ParsedCommand(
        target,
        null,
        annotations,
        overrides
    ),
    systemAttribution,
    forced
);

The dispatch engine then applies its own posture:

final boolean dryRun =
       simulate
    || m.forcedDryRun()
    || !safety.isArmed(rule.automaton());

final ParsedCommand command =
    withPosture(m, dryRun);

final ResolvedSanction resolved =
    resolver.resolve(
        command,
        safety.systemActor().predicate()
    );
  • The descriptor is immutable after construction.
  • Field lookup uses the nominal Type.Rec field order.
  • The command bridge intentionally centralizes validation instead of duplicating sanction policy in the language runtime.
  • Automatic issuance and manual issuance converge at SanctionResolver and SanctionExecutor.
  • The executor marshals entity-touching effects to the Paper main thread.
Complexity
Construction
O(F + C + A), where F is fixed field count, C citation count, and A annotation count.
Materialization
O(C + A).
Resolution
O(A + B), where B is the number of set bits visited during relationship expansion and validation.
Execution
Dominated by asynchronous storage operations and configured delivery effects.
Descriptor storage
O(C + A) before materialization; no descriptor object is persisted directly.

Example

rule<Chat> repeated_flood {
    condition {
        trigger = Trigger::Repeated {
            .count = 5,
            .within = 30s,
            .group_by = GroupBy::Subject,
        };
        guard = (event) ->
            event.length > 120
            && precedent(event.subject, P.2.1) >= 1;
    }
    => verdict = (event) -> Sanction {
        .target = event.subject,
        .cite = { P.2.1 },
        .measure = Measure::MUTE,
        .duration = Duration::Fixed(3h),
        .escalation = Escalation::None,
        .annotations = {
            Annotation::Reason(
                "Repeated message flooding"
            ),
            Annotation::Notify("staff"),
            Annotation::Transcript,
        },
        .attribution = Attribution::System,
        .dry_run = DryRun::InheritSafety,
    };
}
