now

function

Returns the current system wall-clock time as milliseconds since the Unix epoch.

Signature() -> Int
Effectquery
UnitMilliseconds
Origin1970-01-01T00:00:00Z
Clock sourceSystem.currentTimeMillis()
MonotonicNo
CacheNone

Syntax

now()
now-call =
    "now" "(" ")" ;

Parameters

The function takes no parameters.

Constraints

  • The call must contain no arguments.
  • The call is valid only under a query effect allowance.
  • The result has type Int, not Duration.
  • ALAM provides no implicit conversion between epoch timestamps and duration spans.
  • The clock is not guaranteed to increase monotonically.

Semantics

Each call reads the current system wall clock independently. Two calls in one guard may return different values:

now() <= now()

The expression usually evaluates true, but the language does not promise one shared timestamp per event or guard invocation.

Wall-clock behavior

The value follows the host system clock. Administrative clock correction, NTP synchronization, virtualization behavior, or manual clock changes may move it forward or backward.

Use the function for comparisons with stored epoch-millisecond values or fixed configuration deadlines. Do not use subtraction between consecutive calls as a high-resolution monotonic timer.

Relationship to Duration

now() returns an integer timestamp. A Duration is a finite span. Although both are represented by millisecond magnitudes in Java, they are distinct ALAM types.

now()       : Int
30s         : Duration

The current type system does not define timestamp-plus-duration or timestamp-minus-duration operators. A source can compare the timestamp with another Int deadline or perform integer arithmetic when it explicitly stores durations as integer milliseconds.

Dispatch clock distinction

The dispatch engine also reads the guard environment clock before gating candidates. That value is used for trigger windows and throttling. A later now() call inside the guard performs another clock read and need not equal the engine's earlier value.

Effect

query The value changes with mutable external time.

Calling now() from a verdict, constant, transform, matrix cell, trigger expression, expansion argument, or template constraint produces E007.

Implementation notes, non-normative

The built-in signature is:

"now", new Func(
    List.of(),
    INT,
    Effect.QUERY
)

The interpreter returns one integer value:

case "now" ->
    new Value.IntV(f.env.now());

The live environment delegates directly to the JVM wall clock:

@Override
public long now() {
    return System.currentTimeMillis();
}
  • No object or cache entry is retained by the environment.
  • The result fits ALAM's signed 64-bit integer type.
  • The boot-relative BootClock uses System.nanoTime(), but it is unrelated to this ALAM function.
  • Trigger trackers store epoch-millisecond values obtained from the same environment method.

Example

const Int deadline = 1735689600000;

rule<Login> before_deadline {
    condition {
        trigger = Trigger::Atomic;
        guard = (event) ->
            now() < deadline;
    }
    => verdict =
        login_verdict<Measure::CENSURE>;
}
