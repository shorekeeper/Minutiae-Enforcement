ALAM language reference

ALAM is the load-time compiled definition language of the moderation automaton subsystem. A source either produces a complete verified plan or no plan. Runtime dispatch performs no parsing, name resolution, type checking, or structural validation.

Language model

Every expression denotes a typed value. Declarations are processed in source order under declaration-before-use. The type system performs no implicit conversion. Effects form the two-point order pure < query: guards may read the enforcement environment, while verdicts and compile-time declarations remain pure.

The compiler runs lexing, parsing, elaboration, compile-time evaluation, verification, and lowering. Accepted plans contain primitive trigger data plus closed guard and verdict values. Platform events are adapted to ordinary record values before interpretation.

Reference structure

Each entity has one page. Grammar on an entity page covers that entity only. Runtime mechanics are documented on the trigger or function that exposes them. Narrative guides describe the model and pipeline without duplicating entity facts.

Source files and publication

The plugin scans <data>/alam/*.alam in file-name order. Each file is planned independently. A failed file arms no rule and stamps no layout, while accepted siblings remain available. Layout-key collisions across accepted files are deterministic: the first producer in file-name order wins.
