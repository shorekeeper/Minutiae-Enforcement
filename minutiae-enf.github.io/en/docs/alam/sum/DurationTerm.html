  • Duration::Fixed requires exactly one primitive Duration value.
  • A fixed layout duration must be greater than zero.
  • Duration::Permanent is nullary and accepts no argument.
  • In an escalation ladder, a permanent term may appear only as the final rung.
  • The primitive type and descriptor sum share the spelling Duration across disjoint type and value namespaces; they remain different types.

Semantics

Duration::Fixed

Represents an explicitly declared finite span. In a stamped layout, lowering unwraps the primitive value and stores the resulting DurationSpec as the layout duration.

In a runtime-produced sanction, materialization converts the value to the manual override:

duration=<canonical-duration-text>

The canonical text is produced by DurationSpec.format(), for example 1d12h.

Duration::Permanent

Duration::Permanent

The permanent constructor carries no span. Its final meaning depends on the measure:

Containing measureMeaning
Measure::CUSTODY A permanent connection-blocking sanction.
Measure::WARN, Measure::CENSURE, Measure::KICK No explicit duration because the measure is instantaneous.
Temporal measure Ill-formed for a verified layout and insufficient for manual sanction resolution.

The constructor is therefore better understood as the non-fixed term rather than as an unconditional request for permanent enforcement.

Namespace rule

In a type position, Duration denotes the primitive finite-span type. In a value path such as Duration::Fixed, the same root denotes this sum. Type and value namespaces are separate.

Implementation notes, non-normative

The built-in declarations make the namespace distinction explicit:

types.put("Duration", DURATION);          // primitive type namespace
roots.put("Duration", DURATION_TERM);     // constructor path namespace

Layout lowering unwraps only the fixed form:

final DurationSpec duration =
        durationTerm.ctor().equals("Fixed")
            ? ((Value.DurV) durationTerm.field(0)).value()
            : null;

Sanction materialization similarly emits an override only for Fixed:

if (duration.ctor().equals("Fixed")) {
    final DurationSpec d =
            ((Value.DurV) duration.field(0)).value();
    overrides.put("duration", d.format());
}

The shared sanction resolver then interprets the absence of an override from the measure's temporal class:

  • permanent measures resolve to DurationSpec.PERMANENT;
  • instantaneous measures resolve to DurationSpec.ZERO;
  • temporal measures require a finite override or a layout duration.

Example

const Layout temporary_mute = Layout {
    .key = "temporary-mute",
    .rule = P.2.1,
    .reason = "Repeated chat flooding",
    .measure = Measure::MUTE,
    .duration = Duration::Fixed(3h),
    .escalation = Escalation::None,
    .annotations = {},
};

const Layout permanent_custody = Layout {
    .key = "permanent-custody",
    .rule = P.5.1,
    .reason = "Unauthorized client modifications",
    .measure = Measure::CUSTODY,
    .duration = Duration::Permanent,
    .escalation = Escalation::None,
    .annotations = {},
};
