Constraints

Well-formed descriptor requirement

A measure and its duration term must agree on temporal class.

Temporal classMeasuresRequired duration term
Instantaneous Measure::WARN, Measure::CENSURE, Measure::KICK Duration::Permanent, which lowers to absence of an explicit span
Temporal Measure::MUTE, Measure::QUARANTINE, Measure::SUSPENSION Duration::Fixed containing a positive finite Duration
Permanent Measure::CUSTODY Duration::Permanent
  • A Layout violating the temporal table produces R002.
  • A runtime-produced Sanction is materialized onto the command surface and validated by the shared sanction resolver.
  • All constructors are nullary. Applying one with arguments produces an arity or callability diagnostic.
  • Measure is nominal. It is not interchangeable with Text, an annotation name, or the runtime command token.

Semantics

The declaration order defines a total severity order:

WARN < CENSURE < MUTE < KICK < QUARANTINE < SUSPENSION < CUSTODY

The order is used by the sanction resolver to classify a commutation as mitigation or aggravation. It does not automatically select a measure, and it does not imply that every operational deployment must consider adjacent measures equivalent in policy.

Measure::WARN

Measure::WARN

Produces a formal warning. The warning is persisted but does not contribute to the non-warning precedent count used to select escalation rungs. It contributes to the separate warning count used by warn-first policy.

A warn-first gate may replace another resolved measure with Measure::WARN at execution time. In that case the persisted effective measure differs from the measure originally resolved from the ALAM descriptor.

Measure::CENSURE

Measure::CENSURE

Produces an immediate reprimand without a lasting session restriction. Unlike Measure::WARN, it is counted as a non-warning sanction for precedent and escalation.

Measure::MUTE

Measure::MUTE

Applies the MUTED behavioral bit for the resolved duration. Paper chat events are cancelled, and configured command roots are rejected while the bit remains active.

The effect is reconstructed from storage on player join. Overlapping sanctions merge by behavioral bit, with the later expiry retained for each bit.

Measure::KICK

Measure::KICK

Disconnects an online player once. It has no expiry and contributes no persistent behavioral bit. The sanction remains in precedent despite having no active runtime state.

Measure::QUARANTINE

Measure::QUARANTINE

Applies the QUARANTINED behavioral bit for the resolved duration. An online player is moved to the configured anchor and constrained to its configured radius.

The movement listener compares squared distances to avoid square roots. Leaving the configured world triggers confinement to the anchor rather than an ordinary movement rollback.

Measure::SUSPENSION

Measure::SUSPENSION

Blocks connection until the finite expiry. An online non-shadowed subject is disconnected after the sanction is persisted.

Fingerprint signals are captured for connection-blocking measures. Those signals may later contribute to evasion assessment.

Measure::CUSTODY

Measure::CUSTODY

Blocks connection without an expiry. Its permanent temporal class is intrinsic and cannot be changed by a finite duration override.

Shadow exception

A connection-blocking sanction carrying the shadow behavioral bit admits the player at pre-login and applies concealment after join. This preserves the covert-isolation contract instead of revealing the sanction through the refusal screen.

Implementation notes, non-normative

The ALAM sum and runtime enum are intentionally maintained with matching constructor names. Lowering and materialization use those names as the bridge:

final Measure measure =
        Measure.parse(((Value.VariantV) layout.field("measure")).ctor());

The runtime enum stores severity and temporal class directly:

public enum Measure {
    WARN(0, Temporal.INSTANTANEOUS),
    CENSURE(1, Temporal.INSTANTANEOUS),
    MUTE(2, Temporal.TEMPORAL),
    KICK(3, Temporal.INSTANTANEOUS),
    QUARANTINE(4, Temporal.TEMPORAL),
    SUSPENSION(5, Temporal.TEMPORAL),
    CUSTODY(6, Temporal.PERMANENT);
}
  • Measure.parse is case-insensitive on the command and persistence surfaces.
  • SanctionExecutor.behaviourMaskOf maps MUTE and QUARANTINE to behavioral bits.
  • AccessListener queries active SUSPENSION and CUSTODY rows during asynchronous pre-login.
  • Player-facing names are localized by the message service. The enum and ALAM constructor spellings remain canonical and untranslated.

Example

template<Measure M, Duration D>
verdict measured = (event) -> Sanction {
    .target = event.subject,
    .cite = { P.2.1 },
    .measure = M,
    .duration = Duration::Fixed(D),
    .escalation = Escalation::None,
    .annotations = {},
    .attribution = Attribution::System,
    .dry_run = DryRun::InheritSafety,
};

rule<Chat> mute_flood {
    condition {
        trigger = Trigger::Repeated(5, 30s, GroupBy::Subject);
        guard = (event) -> event.length > 120;
    }
    => verdict = measured<Measure::MUTE, 3h>;
}
