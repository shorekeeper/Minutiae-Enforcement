DryRun

sum

The closed sanction-posture type selecting whether a verdict inherits automaton safety configuration or unconditionally suppresses persistence and effects.

Syntax

DryRun::InheritSafety                                    (1)

DryRun::Forced                                           (2)
  • (1)Defers the live-versus-preview decision to automaton arming and simulation posture.
  • (2)Forces the sanction through the dry-run execution path.
dry-run-policy =
      DryRun::InheritSafety
    | DryRun::Forced ;

Constraints

  • The dry_runSanction construction.
  • Both constructors are nullary.
  • Neither constructor can force live execution.
  • DryRun::InheritSafety does not override an unarmed automaton or a simulation request.
  • DryRun::Forced cannot be cancelled by automaton arming.

Semantics

The dispatch engine computes the final dry-run posture as a logical disjunction:

dry_run =
       simulation
    || descriptor_forces_dry_run
    || !automaton_is_armed

DryRun::InheritSafety

DryRun::InheritSafety

Does not request dry-run by itself. A live effect remains possible only when the owning automaton is armed and the firing is not a simulation.

The constructor should be used by ordinary verdicts that are intended to follow deployment configuration. It allows the same compiled definition to run safely in preview before operators arm it.

DryRun::Forced

DryRun::Forced

Marks the descriptor as preview-only. The dispatch engine adds the command-surface @dry-run annotation before resolution.

The resolver still performs ordinary type-independent sanction validation, permission checks, relationship expansion, measure resolution, duration checks, and reason derivation. The executor calculates the effective measure, precedent, warning gate, escalation result, expiry, and behavioral mask, then renders a preview without persistence or player effects.

Dry-run guarantees

A dry-run sanction does not create a sanction row, capture fingerprint signals, persist a chat transcript, activate stays, activate suspended sentences, apply behavioral state, kick a player, publish network directives, broadcast the sanction, or dispatch sanction notifications.

Simulation distinction

The /alam simulate path additionally avoids consuming trigger and safety state. It does not record recurrence occurrences, advance sequence partials, acquire throttle entries, or modify the self-mute set.

A live event evaluated by an unarmed automaton may still consume ordinary trigger state before the resulting sanction is forced into dry-run. Simulation and sanction dry-run are therefore related but not identical concepts.

Implementation notes, non-normative

SanctionMaterializer extracts the descriptor flag:

final Value.VariantV dryRun =
        (Value.VariantV) sanction.field("dry_run");

final boolean forced =
        dryRun.ctor().equals("Forced");

DispatchEngine combines all safety sources:

final boolean dryRun =
       simulate
    || m.forcedDryRun()
    || !safety.isArmed(rule.automaton());

If the final posture is dry-run, the engine adds one token unless already present:

if (dryRun && !present) {
    annotations.add(new RawAnnotation(
        false,
        "dry-run",
        List.of(),
        Map.of()
    ));
}

SanctionExecutor stops before persistence:

if (r.dryRun()) {
    final ExecutionOutcome preview =
        new ExecutionOutcome(
            -1L,
            measure,
            finalDuration,
            expiresAt,
            behaviourMask,
            precedent.priorSanctions(),
            escalated,
            0,
            0,
            r.stayFor() != null,
            !warned && r.suspendFor() != null,
            warned,
            precedent.priorWarnings() + 1,
            r.warnFirst()
        );

    runMain(() -> renderPreview(
        issuer, r, preview, issuedAt, staff
    ));
    return;
}
  • The preview identifier is -1 because no database row exists.
  • Precedent lookup may still perform asynchronous storage access before the preview is rendered.
  • Resolution failures remain visible in dry-run; dry-run is not a validation bypass.

Example

template<Measure M>
verdict preview_only = (event) -> Sanction {
    .target = event.subject,
    .cite = { P.5.1 },
    .measure = M,
    .duration = Duration::Permanent,
    .escalation = Escalation::None,
    .annotations = {},
    .attribution = Attribution::System,
    .dry_run = DryRun::Forced,
};
