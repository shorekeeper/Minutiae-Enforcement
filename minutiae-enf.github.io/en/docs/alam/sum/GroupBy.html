GroupBy

sum

The closed set of partition selectors used to isolate recurrence windows and sequence partial matches.

Syntax

GroupBy::Subject                                           (1)

GroupBy::Global                                            (2)

GroupBy::Field(name)                                       (3)
  • (1)Creates one partition per subject account.
  • (2)Creates one partition shared by all events observed by the rule.
  • (3)Creates one partition per rendered value of a named event fact.
group-by =
      GroupBy::Subject
    | GroupBy::Global
    | GroupBy::Field(field-name) ;

Fields

ConstructorFieldTypeDescription
GroupBy::Subject - - Uses the event subject UUID.
GroupBy::Global - - Uses one constant key for all events.
GroupBy::Field name Text Looks up the named field in incoming event facts and renders its value as text.

Constraints

  • GroupBy::Subject and GroupBy::Global are nullary.
  • GroupBy::Field requires exactly one Text argument.
  • The compiler does not currently prove that a field named by GroupBy::Field exists on every event observed by the containing trigger.
  • For a sequence, every participating step must derive the same partition key for its events to join one partial match.
  • The partition selector does not change event indexing. It only selects state after the rule is already a candidate for the incoming event kind.

Semantics

The dispatch engine creates one opaque tracking key from three components:

automaton-name "|" rule-name "|" partition-value

Including the automaton and rule names prevents state from crossing rule boundaries even when two rules select identical partition values.

GroupBy::Subject

GroupBy::Subject

Uses String.valueOf(event.subject()). For a normal live player event, the result is the canonical UUID text.

If the subject is unavailable, Java renders the partition as the literal text null. All subjectless events observed by that rule therefore share one partition.

GroupBy::Global

GroupBy::Global

Uses the constant partition value *. Every qualifying event observed by the same rule and automaton contributes to one global state machine.

Global recurrence is normally appropriate for server-wide aggregate thresholds. It is not appropriate when one player's activity must not advance another player's sanction trigger.

GroupBy::Field

Reads the raw fact map with EventFacts.field(name) and converts the returned Java object with String.valueOf. The supported live fact values are normally Long, Double, Boolean, and String.

A missing field produces Java null, which is rendered as the partition text null. A missing field and an actual text field whose content is exactly "null" therefore collide.

Sequence partition stability

A sequence stores progress under the key derived from each incoming event. If the selected field is missing, differently named, or has a different value on a later step event, that event addresses another partition and cannot advance the original partial match.

For example, grouping a Login, Chat sequence by the field "ip" is usually invalid operationally: Login exposes ip, while Chat does not. The login enters the IP partition, but the chat event addresses the null partition.

Implementation notes, non-normative

Lowering translates constructors to one small Java record:

private static TriggerPlan.Part partOf(
        final Value.VariantV groupBy) {
    return switch (groupBy.ctor()) {
        case "Subject" -> TriggerPlan.Part.SUBJECT;
        case "Global" -> TriggerPlan.Part.GLOBAL;
        default -> new TriggerPlan.Part(
            TriggerPlan.Part.Kind.FIELD,
            ((Value.TextV) groupBy.field(0)).value()
        );
    };
}

The runtime renders the key as follows:

final String partition = switch (part.kind()) {
    case SUBJECT ->
        String.valueOf(facts.subject());
    case GLOBAL ->
        "*";
    case FIELD ->
        String.valueOf(facts.field(part.field()));
};

return rule.automaton()
    + '|'
    + rule.plan().name()
    + '|'
    + partition;
  • TriggerPlan.Part.SUBJECT and GLOBAL are shared singleton constants.
  • A field selector stores the source field name unchanged.
  • Partition values are not escaped before concatenation. They remain safe as internal opaque keys because the complete string is never parsed back into components.
  • The tracking maps are ConcurrentHashMap instances, providing expected constant-time key lookup.
Complexity
Selection
Expected O(1), including one fact-map lookup for Field.
Key construction
O(A + R + P), proportional to automaton, rule, and partition text lengths.
State lookup
Expected O(1) in the trigger's concurrent partition map.
Storage
No independent selector state; one key string exists for each active trigger partition.

Example

rule<Chat> repeated_message {
    condition {
        trigger = Trigger::Repeated {
            .count = 5,
            .within = 30s,
            .group_by = GroupBy::Subject,
        };
        guard = (event) -> event.length > 120;
    }
    => verdict = flood_verdict<Measure::MUTE, 3h>;
}
