Duration::Fixed

constructor

Wraps one primitive finite Duration value as the fixed-span variant of the descriptor-level duration sum.

Fields

FieldTypeRequiredDescription
value Duration Yes Finite span transported to a layout or sanction duration surface.

Constraints

  • The constructor requires exactly one primitive Duration value.
  • For a temporal layout measure, the span must be greater than zero.
  • For an instantaneous or permanent layout measure, Duration::Fixed violates the measure-duration invariant.
  • For an escalation rung, the span must be greater than zero.
  • The constructor is not itself a primitive Duration and cannot be used as a trigger window.
Type distinction 
3h                         : Duration
Duration::Fixed(3h)        : Duration term

The two values are not assignable to each other. The constructor explicitly crosses from the primitive time-span type to the descriptor sum.

Semantics

The constructor retains the supplied primitive span without changing its magnitude. It does not apply a measure, calculate expiry, consult precedent, or select an escalation rung.

Layout lowering

When used in a verified Layout, lowering unwraps the contained DurationSpec. The layout registry receives that finite duration directly.

The eventual sanction executor computes expiry from the issue timestamp:

expires_at = issued_at + duration.millis

Backdating changes issued_at before expiry is calculated. A backdated sanction may therefore have less remaining time or already be expired when persisted.

Sanction materialization

When returned directly in a Sanction, the materializer converts the span to canonical compact text and writes the command override:

duration=3h
duration=1d12h
duration=2w

The shared resolver reparses that text into its command-side DurationSpec. This round trip intentionally keeps automatic and manual sanction validation on one path.

Escalation rungs

Inside Escalation::Steps, the constructor represents one finite rung. Rung selection returns the contained duration. Probation multiplier mode may then create a larger finite span, and a tariff may impose a floor.

Implementation notes, non-normative

The built-in constructor field is declared against the primitive type:

new Ctor(
    "Fixed",
    List.of(new Field("value", DURATION))
)

Layout lowering unwraps the field directly:

final DurationSpec duration =
    durationTerm.ctor().equals("Fixed")
        ? ((Value.DurV) durationTerm.field(0)).value()
        : null;

Runtime sanction materialization performs a text bridge:

if (duration.ctor().equals("Fixed")) {
    final DurationSpec d =
        ((Value.DurV) duration.field(0)).value();

    overrides.put("duration", d.format());
}

Canonical formatting decomposes from the largest unit to the smallest:

ms = append(sb, ms, WEEK, 'w');
ms = append(sb, ms, DAY, 'd');
ms = append(sb, ms, HOUR, 'h');
ms = append(sb, ms, MIN, 'm');
append(sb, ms, SEC, 's');
  • The internal finite span is measured in milliseconds.
  • Duration arithmetic uses checked multiplication and addition where exposed through DurationSpec.
  • A zero primitive span is representable as 0s, but rejected in verified temporal layouts and escalation ladders.
  • The command resolver independently rejects missing or invalid temporal durations.

Example

const Duration base_span = 3h;

const Layout chat_mute = Layout {
    .key = "chat-mute",
    .rule = P.2.1,
    .reason = "Repeated message flooding",
    .measure = Measure::MUTE,
    .duration = Duration::Fixed(base_span),
    .escalation = Escalation::None,
    .annotations = {},
};
