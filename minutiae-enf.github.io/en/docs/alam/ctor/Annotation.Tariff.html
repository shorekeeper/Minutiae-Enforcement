Annotation::Tariff

constructor

Defines the minimum finite duration that a temporal sanction may retain after base-duration selection, escalation, and probation adjustment.

  • (2)Named-field construction.

Fields

FieldTypeRequiredDescription
floor Duration Yes Minimum finite duration retained by the final sanction calculation.

Constraints

  • The constructor requires exactly one primitive Duration.
  • The resolved measure must have temporal class TEMPORAL.
  • The system actor must hold minutiae.annotation.tariff.
  • A zero floor is valid but has no effect on a non-negative duration.
  • The tariff does not convert an instantaneous or permanent measure into a temporal measure.

Semantics

The executor first derives a base duration from either the explicit duration or the escalation ladder. Probation may alter that base. The tariff is then applied:

if !base.permanent
&& base < tariff_floor:
    final_duration = tariff_floor
else:
    final_duration = base

The comparison uses millisecond magnitude. Equal values remain unchanged.

Calculation order

  1. Resolve the measure.
  2. Reject the tariff when the measure is not temporal.
  3. Select an explicit base duration or an escalation rung.
  4. Apply probation step or multiplier policy.
  5. Apply the tariff floor.
  6. Calculate expiry from effective issue time and final duration.
Permanent base

A permanent escalation rung is never reduced or replaced by a finite tariff. The tariff is a minimum, not a conversion rule.

Explicit duration overrides

An explicit duration bypasses escalation rung selection, but it does not bypass the tariff. A duration shorter than the floor is raised to the floor.

Warning downgrade

When warn-first changes the effective measure to Measure::WARN, the executor calculates the warning as instantaneous. No temporal restriction is applied. The tariff belongs to the originally resolved temporal policy but does not turn the warning into a timed measure.

Implementation notes, non-normative

The resolver rejects non-temporal use before execution:

final DurationSpec tariffFor =
    durationParam(effective, "tariff");

if (tariffFor != null
        && measure.temporal()
            != Measure.Temporal.TEMPORAL) {
    throw new ResolveException(
        MessageKey.RESOLVE_TARIFF_TEMPORAL,
        Arg.s("measure", measure.name())
    );
}

The executor applies the floor after escalation:

if (r.tariffFor() != null
        && !base.permanent()
        && base.millis()
            < r.tariffFor().millis()) {
    base = r.tariffFor();
}
  • The floor is stored in ResolvedSanction.tariffFor.
  • No database column stores the tariff separately.
  • The sanction decision trace stores the final duration but does not currently include a dedicated tariff-applied flag.
  • Materialization uses DurationSpec.format() and the ordinary @tariff validator.

Example

.measure = Measure::SUSPENSION,
.duration = Duration::Fixed(1d),
.escalation = Escalation::Steps({
    Duration::Fixed(12h),
    Duration::Fixed(3d),
    Duration::Fixed(30d),
}),
.annotations = {
    Annotation::Escalate,
    Annotation::Tariff(1d),
},

The first ladder rung is raised from 12 hours to one day. Later rungs already exceed the floor and remain unchanged.
