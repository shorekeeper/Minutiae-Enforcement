Escalation::Steps

constructor

Constructs an explicit ordered escalation ladder whose selected duration is indexed by prior non-warning sanctions.

ContainerEscalation
CategoryConstructor with one list field
PayloadDuration term[]
Compile-time effectpure
Primary consumerStamped Layout
Runtime representationDurationSpec[]
Runtime stateNone

Syntax

Escalation::Steps(steps)                                   (1)

Escalation::Steps {
    .steps = steps,
}                                                           (2)
  • (1)Positional construction.
  • (2)Named-field construction.

Fields

FieldTypeRequiredDescription
steps Duration term[] Yes Ordered duration ladder, from the first-offence outcome to the terminal repeated outcome.

Constraints

Verified layout ladder
  1. The list contains at least one rung.
  2. Every Duration::Fixed rung contains a duration greater than zero.
  3. Duration::Permanent, when present, occurs only as the final rung.
  4. Every item has type Duration term.

The type system verifies item homogeneity. Non-emptiness, positivity, and permanent-rung placement are residual value properties checked later by the verifier.

Verification scope

R003 is currently applied to stamped Layout descriptors. A ladder constructed only inside a runtime Sanction value does not transport its rung array through the command materializer; only the presence of escalation is transported.

Semantics

For a lowered layout ladder of length N and precedent count P, the ordinary selected index is:

index = min(P, N - 1)
Prior non-warning sanctionsSelected rung
0steps[0]
1steps[1], when present
2steps[2], when present
P >= Nsteps[N - 1]

The final rung repeats indefinitely. A ladder does not wrap around and does not produce an out-of-range failure for high precedent counts.

Probation adjustment

When precedent reports an active probation window, runtime configuration selects one of two policies:

PolicyBehavior
Step Select min(P + 1, N - 1).
Multiplier Select the ordinary rung and multiply a finite span by the configured factor.

A permanent rung is not multiplied. It remains permanent.

Overrides and floors

An explicit duration override disables ladder selection for that sanction. After a finite rung is selected, a tariff annotation may raise it to a minimum floor.

if duration_overridden:
    final = explicit_duration
else:
    base = selected_ladder_rung
    final = max(base, tariff_floor)    // finite values only
Postcondition

Rung selection returns one duration value. It does not mutate the ladder or precedent record.

Implementation notes, non-normative

Lowering converts each term to a parallel Java array entry:

final DurationSpec[] ladder =
    new DurationSpec[steps.size()];

for (int i = 0; i < steps.size(); i++) {
    final Value.VariantV rung =
        (Value.VariantV) steps.get(i);

    ladder[i] = rung.ctor().equals("Permanent")
        ? DurationSpec.PERMANENT
        : ((Value.DurV) rung.field(0)).value();
}

The executor selects a rung without scanning the array:

int index = Math.min(
    precedent.priorSanctions(),
    r.escalation().length - 1
);

if (precedent.inProbation()
        && escalationConfig.probationStep()) {
    index = Math.min(
        precedent.priorSanctions() + 1,
        r.escalation().length - 1
    );
}

base = r.escalation()[index];

Multiplier mode creates a new finite DurationSpec:

final long scaled = Math.round(
    base.millis()
        * escalationConfig.probationMultiplier()
);

base = new DurationSpec(scaled, false);
  • Precedent is loaded asynchronously from storage before rung selection.
  • Vacated sanctions are excluded from precedent; pardoned and time-served sanctions remain counted.
  • Sanctions whose decay time has passed are excluded.
  • A decision trace persists the prior count, probation flag, selected ladder index, base duration, and final duration.
