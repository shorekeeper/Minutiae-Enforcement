Trigger::Atomic

constructor

A stateless trigger that presents every candidate event directly to the rule guard.

ContainerTrigger
CategoryNullary trigger constructor
Observed eventsExactly one
Compile-time effectpure
Runtime stateNone
Lowered representationTriggerPlan.Atomic.INSTANCE

Syntax

Trigger::Atomic
atomic-trigger =
    "Trigger" "::" "Atomic" ;

Constraints

Ill-formed if
  1. The containing rule declares other than exactly one event.
  2. The constructor is applied with arguments.
  3. The value is used where a type other than Trigger is required.

The constructor carries no count, span, partition selector, or step list. Adding such state requires Trigger::Repeated or Trigger::Sequence.

Semantics

For each incoming event indexed to the containing rule:

  1. Adapt the event facts to the rule's declared event record.
  2. Invoke the rule guard with that one record.
  3. If the guard returns false, do not fire.
  4. If the guard fails during interpretation, treat the result as false.
  5. If the guard returns true, attempt to acquire the owning automaton's throttle.
  6. If the throttle admits the firing, evaluate the verdict.
State guarantee

An atomic trigger retains no information from one candidate event to the next. Two identical events are evaluated independently.

The absence of trigger state does not mean that the complete firing is side-effect free. A successful firing may consume automaton throttle capacity and may issue a sanction.

Simulation

Simulation evaluates the guard without acquiring the throttle or consulting the automaton self-mute set. A passing guard proceeds through forced dry-run sanction resolution.

Implementation notes, non-normative

The compile-time constructor is a nullary Value.VariantV. Evaluation validates the one-event rule shape:

case "Atomic" -> {
    if (events.size() != 1) {
        error(
            "V001",
            pos,
            "an atomic trigger observes exactly one event; "
                + "the rule declares "
                + events.size()
        );
        throw Abort.INSTANCE;
    }
}

Lowering maps every atomic value to one shared runtime instance:

case "Atomic" ->
    TriggerPlan.Atomic.INSTANCE;

Dispatch uses the same single-event gate as a repeated trigger, with no recurrence plan:

case TriggerPlan.Atomic ignored ->
    gateSingle(rule, facts, now, countFiring, null);

Inside gateSingle, a null repeated plan skips WindowTracker entirely:

if (rep != null) {
    final String key = trackKey(rule, rep.part(), facts);
    if (!windows.record(
            key,
            now,
            rep.count(),
            rep.withinMs())) {
        return Outcome.of(FireDecision.WINDOW_WAIT);
    }
}

return throttleGate(rule)
    ? new Outcome(FireDecision.PASS, args)
    : Outcome.of(FireDecision.THROTTLED);
Complexity
Trigger time
O(1), excluding event adaptation, guard evaluation, and throttle lookup.
Trigger storage
O(1) shared singleton state.
Per partition
No allocation and no retained partition key.

Example

rule<Break> tnt_break {
    condition {
        trigger = Trigger::Atomic;
        guard = (event) -> event.block == "TNT";
    }
    => verdict = break_verdict<P.3.1>;
}
