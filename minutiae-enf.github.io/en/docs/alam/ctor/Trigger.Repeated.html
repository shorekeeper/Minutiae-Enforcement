Trigger::Repeated

constructor

Fires after a guard-admitted event occurs a specified number of times within a trailing span, independently for each partition.

ContainerTrigger
CategoryStateful trigger constructor
Observed eventsExactly one
Compile-time effectpure
Runtime stateOne timestamp ring per active partition
Reset ruleClear the partition ring after a successful recurrence match

Syntax

Trigger::Repeated(count, within, group_by)                    (1)

Trigger::Repeated {
    .count = count,
    .within = within,
    .group_by = group_by,
}                                                               (2)
  • (1)Positional construction.
  • (2)Named-field construction.

Fields

FieldTypeRequiredDescription
count Int Yes Number of admitted occurrences required to fire.
within Duration Yes Trailing finite span containing all required occurrences.
group_by GroupBy Yes Selector producing the independent state partition.

Constraints

Ill-formed if
  1. count is outside the inclusive range 1 through 10000.
  2. within is zero, negative, or permanent.
  3. The containing rule declares other than exactly one event.
  4. A positional construction supplies other than three arguments.
  5. A named construction omits a field, supplies an unknown field, or supplies a field more than once.
  6. A field value has a type different from its declared type.

Semantics

For each incoming candidate event:

  1. Adapt the event facts to the rule's declared event record.
  2. Evaluate the rule guard against that record.
  3. If the guard is false or fails, do not modify the recurrence state.
  4. Derive the partition from group_by.
  5. Append the current epoch-millisecond timestamp to that partition's ring.
  6. Count retained timestamps satisfying the boundary rule.
  7. If at least count timestamps are retained, clear the ring and pass the trigger.
  8. Otherwise, return the runtime decision WINDOW_WAIT.
Boundary rule

An occurrence is in the window if and only if:

timestamp > now - within

An occurrence exactly at the lower boundary is excluded.

Postcondition on match

The matched partition ring is empty. A subsequent firing must accrue another complete set of count admitted occurrences.

Implementation notes, non-normative
  • Lowering produces TriggerPlan.Repeated(int, long, Part).
  • The state key contains automaton name, rule name, and rendered partition value.
  • WindowTracker stores one fixed-capacity long[] ring per key.
  • When full, recording overwrites the oldest timestamp.
  • The partition map is a ConcurrentHashMap.
  • Every 1024 records, an amortized sweep removes empty rings and rings idle for more than one hour.
  • A successful recurrence still must pass the owning automaton's per-minute throttle.
Complexity
Partition lookup
Expected O(1).
Record
O(count), because retained timestamps are scanned after insertion.
Storage
O(count) per active partition.
Sweep
O(P) every 1024 records, where P is the number of tracked partitions.

Example

rule<Chat> mute_flood {
    condition {
        trigger = Trigger::Repeated {
            .count = 5,
            .within = 30s,
            .group_by = GroupBy::Subject,
        };
        guard = (event) -> event.length > 120;
    }
    => verdict = flood_verdict<Measure::MUTE, 3h>;
}
