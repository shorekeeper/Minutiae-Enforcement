diagnostic Parsing

Duplicate field initializer

SeverityError
PhaseParsing
StatusEmitted
Unit outcomeError: unit rejected. Warning: unit may still be published.
Canonical messageduplicate field initializer '.<field>'

Cause

One record construction or with update initializes the same field name more than once.

Field initialization is name-based and has no last-value-wins rule. Duplicate source fields are rejected during parsing.

Example that triggers it

const Layout duplicate = Layout {
    .key = "first",
    .key = "second",
    .rule = P.1.1,
    .reason = "Duplicate key",
    .measure = Measure::CUSTODY,
    .duration = Duration::Permanent,
    .escalation = Escalation::None,
    .annotations = {},
};

Remedy

Remove one of the duplicate initializers.

If a derived value is required, compute the final field expression once or use a separate with expression after constructing the base record.
