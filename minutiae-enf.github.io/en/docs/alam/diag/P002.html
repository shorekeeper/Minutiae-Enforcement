Canonical messageexpected a declaration but found <actual token>

Cause

A token appears at the top level where ALAM requires the beginning of a declaration.

Expressions and automaton items cannot appear directly in the compilation unit. Top-level source must begin with a declaration keyword.

Example that triggers it

Trigger::Atomic;

const Int threshold = 5;

Remedy

Place the expression inside an appropriate declaration.

Begin the top-level construct with const, schema, matrix, transform, expand, template, rule, or automaton.

Check whether an earlier declaration closed too early and left part of its body at top level.
