Canonical messageexpected an expression but found <actual token>

Cause

The parser entered an expression position, but the current token cannot begin any ALAM primary expression.

Valid expression starts include literals, identifiers, true, false, an opening parenthesis, an opening brace, and match.

Example that triggers it

const Int threshold = ;

Remedy

Supply the missing expression.

If the expression was intended to occur before the reported token, check for a misplaced comma, semicolon, assignment token, or closing delimiter.

Do not use declaration keywords where a value expression is required.
