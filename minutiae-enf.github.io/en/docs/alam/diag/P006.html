Path segment requires a name path

SeverityError
PhaseParsing
StatusEmitted
Unit outcomeError: unit rejected. Warning: unit may still be published.
Canonical message'::' requires a name path on its left

Cause

The path separator :: follows an expression that is neither a bare name nor an existing path.

Constructor and named-value paths are syntactic name paths. They cannot be appended to a call result, record value, literal, member value, or other arbitrary expression.

Example that triggers it

const Measure value = make_measure()::MUTE;

Remedy

Place :: after a name or an existing name path, for example Measure::MUTE.

If field access was intended, use a dot and ensure the receiver is a record or event value.

If the left expression was intended to return a constructor or namespace, rewrite the source to name the constructor directly.
