Wildcard is not an expression

SeverityError
PhaseParsing
StatusEmitted
Unit outcomeError: unit rejected. Warning: unit may still be published.
Canonical messagethe wildcard is not an expression

Cause

The standalone underscore token appears where a value expression is required.

The wildcard is admitted only in match patterns and template specialization arguments. It does not denote a runtime or compile-time value.

Example that triggers it

const Int value = _;

Remedy

Replace the wildcard with an expression of the required type.

Use _ only as a match catch-all pattern or as a template specialization wildcard.
