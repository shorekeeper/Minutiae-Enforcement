Expected template parameter name

SeverityError
PhaseParsing
StatusEmitted
Unit outcomeError: unit rejected. Warning: unit may still be published.
Canonical messageexpected a template parameter name

Cause

A primary template parameter contains a type but no following identifier.

Each primary template parameter is a type-name pair. The closing angle bracket or comma was encountered where the parameter name is required.

Example that triggers it

template<Measure>
verdict policy = (event) -> Sanction {
    .target = event.subject,
    .cite = { P.1.1 },
    .measure = Measure::CUSTODY,
    .duration = Duration::Permanent,
    .escalation = Escalation::None,
    .annotations = {},
    .attribution = Attribution::System,
    .dry_run = DryRun::InheritSafety,
};

Remedy

Add an identifier after every template parameter type, for example template<Measure M>.

For several parameters, separate complete type-name pairs with commas.
