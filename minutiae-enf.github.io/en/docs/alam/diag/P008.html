Canonical messageexpected a pattern: integer, range, path, or '_'

Cause

A match arm begins with a token that cannot form an ALAM pattern.

Valid patterns are an integer literal, an inclusive integer range, a constructor path, or the wildcard underscore.

The same code is also used when the upper bound after .. is not an integer literal.

Example that triggers it

const Int value = match(3) {
    "three" => 3,
    _ => 0,
};

Remedy

Use a pattern compatible with the subject type.

For an Int subject, use an integer, an inclusive integer range, or _.

For an enumerated nullary sum, use a constructor path such as Measure::MUTE or _.

Ensure both bounds of an integer range are integer literals.
