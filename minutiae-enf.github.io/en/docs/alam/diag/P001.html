Canonical messageexpected <token or construct> but found <actual token>

Cause

The parser reached a position where the grammar requires a specific token, name, delimiter, type, clause, or declaration component, but the current token does not satisfy that requirement.

P001 is the general structural parser diagnostic. The complete log message names the expected construct and the token actually found.

Example that triggers it

const Int threshold = 5

Remedy

Read the complete diagnostic message and insert or replace the named token.

Check delimiters immediately before the reported position. A missing semicolon, comma, closing brace, or closing parenthesis often shifts the failure onto the next otherwise valid token.

Compare the declaration with the Syntax section of the referenced entity page.
