Invalid decoded duration value

Cause

The parser received a DURATION token, but DurationSpec rejected its decoded value.

This is a defensive parser diagnostic. Under the current lexer, a token satisfying the strict duration-token grammar also satisfies the duration decoder for all recognized units, so no ordinary source example is known to reach this code.

The emission remains in place to preserve a positioned parser failure if lexer and decoder rules diverge in a future version.

Example that triggers it

No triggering example is defined for this diagnostic.

Remedy

When this diagnostic is produced, use a simple canonical duration made from integer-unit segments such as 30s, 3h, 7d, or 1w2d.

Check that the documentation, lexer, and runtime belong to the same plugin version.

Report the exact literal and compiler version if a lexer-accepted duration reaches this diagnostic.
