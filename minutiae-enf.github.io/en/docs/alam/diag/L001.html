L001

diagnostic Lexing

Unexpected character

SeverityError
PhaseLexing
StatusReserved
Unit outcomeError: unit rejected. Warning: unit may still be published.
Canonical messageunexpected character '<character>'

Cause

The lexer encountered a character that does not begin any ALAM token.

The offending character is emitted as an ERROR token. Lexing then resumes at the following character so later source positions remain diagnosable.

Example that triggers it

const Int value = 1 @ 2;

Remedy

Remove the unsupported character or replace it with the intended ALAM operator or punctuation.

If the character belongs to text data, place it inside a double-quoted text literal.

Compiler emission sites

Implementation notes, non-normative

This code is not emitted by the current compiler.

Line numbers are generated from the current Java source and may change without changing the diagnostic contract.
