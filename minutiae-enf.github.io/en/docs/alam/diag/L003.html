L003

diagnostic Lexing

Invalid text escape

SeverityError
PhaseLexing
StatusEmitted
Unit outcomeError: unit rejected. Warning: unit may still be published.
Canonical messageinvalid escape '\<character>'

Cause

A backslash inside a text literal is followed by a character outside the supported escape set.

The supported escapes are double quote, backslash, n, t, and r. The lexer reports the unsupported escape and continues scanning the surrounding text literal.

Example that triggers it

const Text path = "bad\qescape";

Remedy

Use one of the supported escape sequences: \" for a quote, \\ for a backslash, \n for newline, \t for tab, or \r for carriage return.

Remove the backslash when the following character is intended literally.
