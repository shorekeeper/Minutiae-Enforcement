Expansion requires an application

SeverityError
PhaseParsing
StatusEmitted
Unit outcomeError: unit rejected. Warning: unit may still be published.
Canonical message'expand' requires a transform application of the form name(argument)

Cause

The expression following expand is not syntactically a call expression.

An expansion must identify the transform and matrix argument in one application so elaboration can verify the single-parameter mapping.

Example that triggers it

expand layout_rows;

Remedy

Call the transform with the matrix expression, for example expand make_layout(layout_rows);

Ensure the expansion ends with a semicolon.

After fixing the syntax, resolve any E023 diagnostic concerning transform purity, arity, input type, or result type.
