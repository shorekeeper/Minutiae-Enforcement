L005

diagnostic Lexing

Integer literal overflow

SeverityError
PhaseLexing
StatusEmitted
Unit outcomeError: unit rejected. Warning: unit may still be published.
Canonical messageinteger literal exceeds the signed 64-bit range

Cause

The positive magnitude of an integer literal is greater than Long.MAX_VALUE.

Integer digits are accumulated during lexing with overflow detection. The token is rejected before parsing or type checking.

Example that triggers it

const Int value = 9223372036854775808;

Remedy

Use a value in the signed 64-bit range.

If the value is intended as text or an external identifier rather than an arithmetic integer, store it in Text.
