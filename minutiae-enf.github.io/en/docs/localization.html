Localization

Every player-facing line renders in the recipient's own locale - including a broadcast, where each viewer sees their own language. This page covers how bundles resolve, how to translate, and how grammatical forms work.

Bundles

A bundle is a YAML file at lang/<tag>.yml - en.yml, ru.yml, uk.yml. Templates use ampersand colour codes plus hex (&#8AFFEB) and {name} placeholders:

lift:
  success: "{prefix}&#79FFB0Lifted &#E8EEF0#{id} &8({kind}) ..."
{prefix} Injected from the bundle's own prefix entry, colour codes intact.
scalar values Player names, reasons, and other user-supplied content are stripped of colour codes before substitution - a reason cannot inject formatting.
rich values Some placeholders ({measure}, {duration}, {rank}, {kind}, {reason}) are rendered per recipient at delivery, which is what makes a mixed-language broadcast work. The reason placeholder additionally carries a hover card.
multi-line A YAML list joins with newlines into one message - the ban screen and help text use this.

How a message resolves

For each key independently, the first layer that defines it wins:

  1. The on-disk bundle for the player's exact locale tag (ru_ru);
  2. the on-disk bundle for the language part (ru);
  3. the on-disk bundle for messages.default-locale;
  4. the bundle for the default locale shipped inside the jar;
  5. the shipped English bundle.
Note

Layers 4 and 5 exist because on-disk files are provisioned once and never overwritten. A plugin update that introduces new message keys cannot edit your files - the shipped bundle fills the gap until you merge. The boot log tells you when a merge is available: bundle 'ru' lacks 10 key(s); embedded defaults apply.

messages.per-player-locale: false collapses steps 1-2 and serves everyone the default locale. The console, logs, and the pre-login refusal screen always use the default locale - a player's language is not known before login completes.

Measure and kind names

Enum identifiers (MUTE, CUSTODY) appear in commands and configuration only; players see display names from the bundle:

measure:
  mute: "Mute"
  suspension: "Temporary Ban"
  custody: "Permanent Ban"

lift:
  kind:
    vacate: "vacated"
    pardon: "pardoned"
    time-served: "time served"

Durations are localised the same way, through unit suffixes:

duration:
  permanent: "permanent"
  unit: { weeks: "w", days: "d", hours: "h", minutes: "m", seconds: "s" }

Rank names and grammatical forms

Rank identifiers come from your configuration; their display names live in the bundle under rank.<id>. Because ranks are operator-defined, this namespace is open - any rank.* path you add is picked up without a plugin change.

Inflected languages need more than one form. A template reading "sanctioned by {rank}" wants the genitive in Russian, so the placeholder requests the by form, which resolves rank.<id>.by and falls back to the base entry when absent:

# en.yml - no grammatical case, base entries only:
rank:
  moderator: "&#8AFFEBModerator"

# ru.yml - base plus the agentive form:
rank:
  moderator: "&#8AFFEBМодератор"
  moderator.by: "&#8AFFEBмодератора"

English bundles simply never define the forms; every request collapses onto the base entry. The same fallback applies to any form suffix a future template introduces.

Translating to a new language

  1. Copy lang/en.yml to lang/<tag>.yml - the tag is the lowercased client language code (de, pl, uk).
  2. Translate values. Never translate keys, placeholder names, or the measure identifiers used in config.yml.
  3. /me-ban reload. Bundles reload live.
  4. Missing keys are fine: they fall through to the default locale and then the shipped bundles. Translate incrementally; nothing renders as a raw placeholder as long as the shipped English bundle knows the key.
Warning

Keep placeholders intact, including braces: {target} translated as {цель} will print literally. A quick check after translating: trigger each message once on a test server and look for braces in chat.

What is not localised

Deliberately, and worth knowing before you file it as a bug:

Command vocabulary ::layout keys, @annotation names, measure identifiers, confirm - grammar, not text.
Server log output Always English: logs are operator diagnostics and get pasted into issue trackers.
Persisted strings Reasons, audit details, and amendment diffs are stored as written and shown as stored - a string in the database has no per-viewer locale.
The web panel English-only in the current release.
