Web panel

A built-in case browser styled after a registry editor: subjects, sanctions, appeals, transcripts, and audit trails as one navigable tree. No external web server, no database exposure.

Setup

web:
  enabled: true
  bind: 127.0.0.1
  port: 8787
  tokens:
    - token: "generate-something-long-and-random"
      role: read
    - token: "another-long-random-value"
      role: write

Restart after changing this section. The panel is then at http://127.0.0.1:8787/; the login form takes a token and stores it as a session cookie.

Important

Replace both placeholder tokens before enabling, and treat them as passwords - a token is the entire credential. Keep bind on loopback and put TLS in front; see deployment.

Roles and authority

read Navigation and search only. Cannot change anything.
write Read, plus: lift a sanction, accept or deny an appeal, and issue a sanction through the same grammar as the in-game command.

A write token is deliberately weaker than an administrator. Panel actions execute through a synthetic actor holding a fixed, bounded grant set - the everyday annotation surface plus foreign and blocking lifts. It can never:

  • impersonate staff (@as);
  • aggravate a commutation;
  • amend existing sanctions;
  • backdate;
  • bulk-lift or reverse lifts.

A compromised write token is a moderator, not root. Token comparison is constant-time, so the token space cannot be probed through response timing.

The tree

Five top-level hives, each a concern:

HKEY_CASES The sanction tree, descending subject, measure, enforcer, sanction. Provenance is the hierarchy itself: to see who sanctioned whom, you descend, never chase cross-references. Each sanction opens into provisions, captured signals, transcript, rationale, amendments, woven children, and its audit trail.
HKEY_APPEALS Appeals by appellant and in aggregate, with pending ones actionable under a write token.
HKEY_AMENDS Amendments and per-sanction traceback histories.
HKEY_FINGERPRINTS The forensic network: signal values and their bearers, account clusters, and per-suspect evidence decomposition. See Evasion detection.
HKEY_SYSTEM Aggregate counters, uptime, schema version, and the global audit trail.

Worth knowing while navigating:

  • The Rationale node under a sanction shows the recorded decision trace: prior counts consulted, ladder rung selected, probation state, warn-first downgrade. It answers "why 30 days?" without recomputation.
  • The Transcript node opens captured chat as a CSV table in a modal.
  • Search (top right) takes a sanction id or a player name.
  • Arrow keys navigate the tree; Backspace-style Up, Back, and Forward buttons behave like a file manager.

Write actions

Lift On any unlifted sanction node. Prompts for a reason; executes as a vacatur through the same audited path as the command.
Accept / Deny On any pending appeal. Acceptance vacates the underlying sanction.
Manage Issues a sanction from a spec string in the command grammar: Player ::toxic or Player @measure(MUTE) duration=1h. The spec passes the full resolver - validation errors come back verbatim.

Deployment behind a reverse proxy

The built-in server speaks plain HTTP and is designed to sit behind a TLS-terminating proxy on the same host:

server {
    listen 443 ssl;
    server_name panel.example.org;
    # ... ssl_certificate ...

    location / {
        proxy_pass http://127.0.0.1:8787;
        proxy_set_header Host $host;
        allow 203.0.113.0/24;   # your staff network, if applicable
        deny all;
    }
}
  • Keep bind: 127.0.0.1 - only the proxy reaches the panel directly.
  • The session cookie is HttpOnly and SameSite=Strict; on a non-loopback bind the Secure attribute is added, which is one more reason not to expose plain HTTP.
  • Rotate tokens by editing the list and restarting; a removed token dies with the restart.

Limitations

English only The panel UI is not localised in this release.
Vacate only The lift button always vacates. Pardons and time-served lifts are issued in game, where the full lift grammar is available.
Read-mostly by design Configuration, layouts, rules, and bundles are files - the panel neither shows nor edits them.
