MEMinutiae Enforcement docs

Configuration

A guided tour of config.yml, section by section. Rules and layouts live in their own files and have their own pages; this page covers everything else.

Note

Most sections apply live through /me-ban reload. Sections that own physical resources - storage, the web panel bind address - require a restart; each is marked below.

storage

storage:
  driver: SQLITE
  sqlite:
    file: minutiae.db
  pool:
    max-connections: 4
    acquire-timeout-ms: 5000
    busy-timeout-ms: 5000
driver Backend selector. SQLITE is the only serviceable value; MYSQL is reserved.
sqlite.file Database path, relative to the plugin folder unless absolute.
pool.max-connections Physical connections held by the pool. Four is right for almost everyone; SQLite serialises writes internally, so more connections help reads only.
pool.acquire-timeout-ms How long a borrower waits before failing deterministically instead of hanging.
pool.busy-timeout-ms SQLite busy_timeout per connection.

Requires a restart. Schema migrations run automatically at boot and are forward-only; downgrading the plugin against a newer database is not supported.

messages

messages:
  default-locale: en
  per-player-locale: true
  reason:
    abbreviate: 32
default-locale Bundle used for the console, pre-login screens, and any player whose own locale has no bundle.
per-player-locale When true, each player receives messages in their client language, falling back per key. When false, everyone gets the default locale.
reason.abbreviate Characters of a reason shown inline before it is cut with an ellipsis; the full text stays in the hover card.

How bundles resolve, and how to translate, is covered in Localization.

broadcast

broadcast:
  enabled: true
  measures: [SUSPENSION, CUSTODY, KICK, MUTE, QUARANTINE]
  lift: true
enabled Master switch for public announcements.
measures Measures announced when applied. Warnings, stayed, and dry-run sanctions are never broadcast regardless.
lift Whether lifted sanctions are announced. A single lift can opt out with @silent.

Broadcasts render per viewer: each player sees the measure name, duration, and staff rank in their own language. @silent on a sanction (and @shadow, which implies it) suppresses its announcement.

ranks

ranks:
  default: staff
  console: console
  entries:
    - { id: administrator, permission: minutiae.rank.administrator }
    - { id: moderator, permission: minutiae.rank.moderator }
    - { id: helper, permission: minutiae.rank.helper }

Attributes broadcasts and notifications to the issuer's role through the {rank} placeholder, so an announcement reads "sanctioned by Moderator Steve" rather than assuming everyone is generic staff.

entries Scanned in order; the first permission the issuer holds wins. List senior ranks first, since senior staff usually hold junior nodes too.
default Identifier used when nothing matches - including synthetic issuers such as automation and the web panel.
console Identifier for the console, resolved without a scan (the console holds every node and would always match the first entry).

The minutiae.rank.* nodes are markers, not capabilities - they gate nothing. Display names, including grammatical forms for inflected languages, live in the language bundles under rank.<id>; see Localization.

notify

notify:
  default-channels: [staff]
  channels:
    staff:
      type: STAFF
      permission: minutiae.notify.staff
    console:
      type: CONSOLE
    audit:
      type: LOG
    # discord:
    #   type: WEBHOOK
    #   url: "https://discord.com/api/webhooks/xxxxx/yyyyy"

Named delivery channels consumed by @notify(channel...) and by system events (new appeals, due reviews, lift notifications).

STAFF In-game broadcast to online holders of the channel's permission, localised per recipient.
CONSOLE A line to the server console.
LOG A line to the plugin logger.
WEBHOOK A plain-text JSON payload posted to an HTTP endpoint. The shape suits Discord webhooks directly.
default-channels Used when a system event fires without explicit channels.

A reference to an undefined channel is reported once in the log and skipped; it never fails the sanction.

behaviour

behaviour:
  quarantine:
    world: world
    x: 0.5
    y: 100.0
    z: 0.5
    radius: 16.0
  mute:
    blocked-commands: [msg, tell, w, me, say, r, reply, whisper]
  rubberband:
    leash: 3.0
quarantine Anchor and confinement radius for the QUARANTINE measure and @remand. A missing world falls back to the primary world with a logged warning.
mute.blocked-commands Command roots suppressed while muted, in addition to chat. Compared against the leading token without the slash.
rubberband.leash Maximum per-move displacement in blocks before the movement is reverted, for @rubberband.

escalation

escalation:
  probation-mode: STEP
  probation-multiplier: 2.0

Governs how an active probation window (opened by @probation at issue or lift time) aggravates a repeat offence under the same rule:

STEP The escalation ladder advances one extra rung.
MULTIPLIER The computed duration is scaled by probation-multiplier instead.

Ladders themselves are declared per layout; see Layouts and escalation.

chat-history

chat-history:
  enabled: true
  mode: ANNOTATION
  per-player: 50
  max-message-length: 256
  max-tracked: 512
  retention-grace: 900

A bounded ring of recent chat lines is kept per player; a snapshot is persisted with a sanction when requested and viewable as CSV in the web panel.

mode ANNOTATION captures only for sanctions carrying @transcript; ALWAYS additionally captures for bans and behavioural sanctions.
per-player / max-message-length / max-tracked Memory bounds: ring depth, stored characters per line, and players tracked at once. Every bound exists to neutralise abuse by sustained chat.
retention-grace Seconds a departed player's ring is kept, so a sanction issued just after a rage-quit still captures evidence.

fingerprint

Evasion detection has enough knobs to warrant its own page: thresholds, the evidence model, network classification, and clustering are covered in Evasion detection. The one switch worth knowing here:

fingerprint:
  enabled: true

web

web:
  enabled: false
  bind: 127.0.0.1
  port: 8787
  tokens:
    - token: "change-me-read"
      role: read
    - token: "change-me-write"
      role: write

Disabled and loopback-bound by default; change both tokens before enabling. Roles, authority bounds, and reverse-proxy deployment are covered in Web panel. Requires a restart.

alam

alam:
  armed: false
  armed-automata: []
  throttle:
    per-minute: 30
  system:
    name: SYSTEM
    allow-all: false
    permissions:
      - minutiae.command.enforce
      - minutiae.annotation.measure
      # ...

The automation engine's safety posture. An automaton that is not armed evaluates in forced dry-run: it resolves and reports but applies nothing. The system block bounds what automation may do at all - it is a permission grant, so automation can never exceed a manual command issued by a moderator holding the same set. The definition language itself is documented separately.

maintenance and boot

maintenance:
  interval: 30
  session-grace: 300
boot:
  verbose: true
maintenance.interval Seconds between background sweeps: expired behavioural records, stale fingerprint sessions, concealment reconciliation.
maintenance.session-grace Seconds a disconnected player's fingerprint session is retained, so a ban issued right after disconnect still captures signals.
boot.verbose Per-stage boot diagnostics with timings. Costs nothing after boot; leave it on.
