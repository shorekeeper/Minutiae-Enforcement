MEMinutiae Enforcement docs

Getting started

From a jar file to your first audited sanction. Nothing here requires a database server or an external dependency.

Requirements

Server Paper 1.21 or newer. Spigot is not supported: the plugin uses Paper's command and chat APIs directly.
Java Java 21 or newer.
Storage None to install. The plugin ships with SQLite and creates its database file on first boot.

Installation

  1. Drop MinutiaeEnforcement.jar into your server's plugins/ directory.
  2. Start the server. The boot log prints one line per subsystem; a healthy boot ends with ready: all subsystems online.
  3. Stop the server if you want to edit the generated configuration before real use, or keep going with the defaults.

First boot creates the following files:

config.yml Runtime configuration. Every section is documented inline; see Configuration for the guided tour.
rules.yml Your server rules as numbered entries. Ships with a nine-rule example set; replace it with your own.
layouts.yml Sanction templates. Ships with examples covering griefing, cheating, and chat toxicity.
lang/en.yml The message bundle for your default locale. Edited files are never overwritten; keys you have not merged yet fall back to the shipped defaults.
minutiae.db The SQLite database. Sanctions, appeals, audit, chat transcripts, and fingerprint data live here.
alam/ Automation definitions. Empty by default; ignore it until you want automatic enforcement.

Verify the installation

/me-ban ping

This probes the storage backend and reports round-trip latency and the schema version. If it answers, everything below it works.

Grant permissions

Two bundle nodes cover almost everything:

minutiae.staff The day-to-day moderation surface: issuing, lifting, dockets, appeals review, most annotations.
minutiae.admin Everything, including bulk lifts, lift reversal, reloads, and the destructive annotations. Granted to operators by default.
Note

Leaf nodes register with default: false, which applies to operators too. An operator holds a leaf only through the minutiae.admin chain. If a command disappears for an operator, the chain in paper-plugin.yml or your permissions plugin is broken; see Permissions.

With LuckPerms:

/lp group moderator permission set minutiae.staff true

Your first sanction

Using a shipped example layout:

/enforce SomePlayer ::toxic

The toxic layout cites rule P.2.4, applies a three-hour mute, and carries @warn-first(2): the first two offences under this rule become formal warnings instead of mutes. Run it twice on a test account and watch the report change.

Fully manual, without a layout:

/enforce SomePlayer @measure(KICK) reason="Testing, ignore"

Not sure yet? Preview without applying anything:

/enforce SomePlayer ::griefing @dry-run

Undoing it

Every report ends with an ID line. Lift by that identifier:

/enforce lift 1 vacate "Test sanction"

vacate erases the sanction from the player's precedent, which is what you want for a test. The distinction between vacate, pardon, and time served is covered in Lifting sanctions.
