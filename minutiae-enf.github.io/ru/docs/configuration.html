MEMinutiae Enforcement docs

Конфигурация

Экскурсия по config.yml, секция за секцией. Правила и шаблоны живут в собственных файлах и имеют собственные страницы; эта покрывает всё остальное.

Примечание

Большинство секций применяется вживую через /me-ban reload. Секции, владеющие физическими ресурсами — хранилище, адрес веб-панели, — требуют рестарта; каждая помечена ниже.

storage

storage:
  driver: SQLITE
  sqlite:
    file: minutiae.db
  pool:
    max-connections: 4
    acquire-timeout-ms: 5000
    busy-timeout-ms: 5000
driver Селектор бэкенда. SQLITE — единственное рабочее значение; MYSQL зарезервирован.
sqlite.file Путь к базе, относительный к папке плагина, если не абсолютный.
pool.max-connections Физические соединения пула. Четырёх достаточно почти всем; SQLite сериализует записи внутренне, так что добавка соединений помогает только чтениям.
pool.acquire-timeout-ms Сколько заёмщик ждёт, прежде чем детерминированно отказать вместо зависания.
pool.busy-timeout-ms SQLite busy_timeout на соединение.

Требует рестарта. Миграции схемы выполняются автоматически при запуске и только вперёд; даунгрейд плагина против более новой базы не поддерживается.

messages

messages:
  default-locale: ru
  per-player-locale: true
  reason:
    abbreviate: 32
default-locale Пакет для консоли, экранов до входа и любого игрока, для чьей локали пакета нет.
per-player-locale При true каждый игрок получает сообщения на языке своего клиента, с поключевым фолбэком. При false все получают локаль по умолчанию.
reason.abbreviate Символы причины, показываемые в строке до обрезки многоточием; полный текст остаётся во всплывающей карточке.

Как разрешаются пакеты и как переводить — на странице Локализация.

broadcast

broadcast:
  enabled: true
  measures: [SUSPENSION, CUSTODY, KICK, MUTE, QUARANTINE]
  lift: true
enabled Главный выключатель публичных объявлений.
measures Меры, объявляемые при применении. Предупреждения, отложенные и пробные санкции не объявляются никогда.
lift Объявлять ли снятые санкции. Одиночное снятие может отказаться через @silent.

Объявления рендерятся по зрителям: каждый игрок видит название меры, срок и ранг персонала на своём языке. @silent на санкции (и @shadow, который его влечёт) подавляет её объявление.

ranks

ranks:
  default: staff
  console: console
  entries:
    - { id: administrator, permission: minutiae.rank.administrator }
    - { id: moderator, permission: minutiae.rank.moderator }
    - { id: helper, permission: minutiae.rank.helper }

Атрибутирует объявления и уведомления роли выдавшего через плейсхолдер {rank}, чтобы объявление читалось «санкция от модератора Steve», а не предполагало, что все — безликий персонал.

entries Сканируются по порядку; побеждает первое право, которое держит выдавший. Старшие ранги — первыми, поскольку старший персонал обычно держит и младшие узлы.
default Идентификатор, когда ничего не совпало — включая синтетических выдающих вроде автоматизации и веб-панели.
console Идентификатор консоли, разрешаемый без сканирования (консоль держит каждый узел и всегда совпала бы с первой записью).

Узлы minutiae.rank.* — маркеры, не возможности: они ничего не открывают. Отображаемые имена, включая грамматические формы для флективных языков, живут в языковых пакетах под rank.<id>; см. Локализацию.

notify

notify:
  default-channels: [staff]
  channels:
    staff:
      type: STAFF
      permission: minutiae.notify.staff
    console:
      type: CONSOLE
    audit:
      type: LOG
    # discord:
    #   type: WEBHOOK
    #   url: "https://discord.com/api/webhooks/xxxxx/yyyyy"

Именованные каналы доставки, потребляемые @notify(канал...) и системными событиями (новые апелляции, подошедшие пересмотры, уведомления о снятии).

STAFF Внутриигровая рассылка держателям права канала онлайн, локализованная по получателям.
CONSOLE Строка в консоль сервера.
LOG Строка в логгер плагина.
WEBHOOK Текстовый JSON, отправляемый POST-ом на HTTP-адрес. Форма подходит вебхукам Discord напрямую.
default-channels Используются, когда системное событие срабатывает без явных каналов.

Ссылка на неопределённый канал сообщается в лог один раз и пропускается; санкцию она не роняет никогда.

behaviour

behaviour:
  quarantine:
    world: world
    x: 0.5
    y: 100.0
    z: 0.5
    radius: 16.0
  mute:
    blocked-commands: [msg, tell, w, me, say, r, reply, whisper]
  rubberband:
    leash: 3.0
quarantine Якорь и радиус заключения для меры QUARANTINE и @remand. Отсутствующий мир заменяется основным с предупреждением в логе.
mute.blocked-commands Корни команд, блокируемые в муте вдобавок к чату. Сравниваются с ведущим токеном без слэша.
rubberband.leash Максимальное смещение за ход в блоках, после которого перемещение откатывается, — для @rubberband.

escalation

escalation:
  probation-mode: STEP
  probation-multiplier: 2.0

Определяет, как активное окно пробации (открытое @probation при выдаче или снятии) отягчает рецидив по тому же правилу:

STEP Лестница эскалации продвигается на одну лишнюю ступень.
MULTIPLIER Вычисленный срок умножается на probation-multiplier.

Сами лестницы объявляются по шаблонам; см. Шаблоны и эскалацию.

chat-history

chat-history:
  enabled: true
  mode: ANNOTATION
  per-player: 50
  max-message-length: 256
  max-tracked: 512
  retention-grace: 900

Ограниченное кольцо недавних строк чата хранится по игрокам; снимок сохраняется с санкцией по запросу и просматривается как CSV в веб-панели.

mode ANNOTATION захватывает только для санкций с @transcript; ALWAYS дополнительно — для банов и поведенческих санкций.
per-player / max-message-length / max-tracked Границы памяти: глубина кольца, символы на строку и число игроков в отслеживании. Каждая граница существует, чтобы нейтрализовать злоупотребление непрерывным чатом.
retention-grace Секунды хранения кольца вышедшего игрока — чтобы санкция сразу после гневного выхода всё ещё захватила улики.

fingerprint

У детекта обхода достаточно ручек для собственной страницы: пороги, модель улик, классификация сетей и кластеризация — в Детекте обхода банов. Единственный переключатель, который стоит знать здесь:

fingerprint:
  enabled: true

web

web:
  enabled: false
  bind: 127.0.0.1
  port: 8787
  tokens:
    - token: "change-me-read"
      role: read
    - token: "change-me-write"
      role: write

По умолчанию выключена и привязана к loopback; смените оба токена до включения. Роли, границы полномочий и развёртывание за обратным прокси — на странице Веб-панель. Требует рестарта.

alam

alam:
  armed: false
  armed-automata: []
  throttle:
    per-minute: 30
  system:
    name: SYSTEM
    allow-all: false
    permissions:
      - minutiae.command.enforce
      - minutiae.annotation.measure
      # ...

Позиция безопасности движка автоматизации. Невзведённый автомат вычисляется в принудительном dry-run: разрешает и сообщает, но ничего не применяет. Блок system ограничивает, что автоматизация вообще может, — это грант прав, так что автоматизация никогда не превысит ручную команду модератора с тем же набором. Сам язык определений документируется отдельно.

maintenance и boot

maintenance:
  interval: 30
  session-grace: 300
boot:
  verbose: true
maintenance.interval Секунды между фоновыми проходами: истёкшие поведенческие записи, устаревшие сессии отпечатков, сверка сокрытия.
maintenance.session-grace Секунды хранения сессии отпечатков отключившегося игрока — чтобы бан сразу после выхода всё ещё захватил сигналы.
boot.verbose Постадийная диагностика запуска с таймингами. После запуска не стоит ничего; оставьте включённой.
