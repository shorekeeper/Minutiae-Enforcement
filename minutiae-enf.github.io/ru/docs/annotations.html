Справочник аннотаций

Аннотация — токен вида @name или @name(параметры), прикреплённый к санкции. Эта страница перечисляет каждую встроенную аннотацию с её параметрами, областью применения и узлом прав.

Синтаксис

@flag                      без параметров
@name(value)               один позиционный параметр
@notify(staff, discord)    несколько позиционных параметров
@evidence(url="https://example.org/clip", type=video)
!@escalate                 отрицание: убирает аннотацию из шаблона

Значения с запятыми или пробелами берутся в двойные кавычки. Отрицание (!@name) действительно только в команде и убирает только аннотации, пришедшие из выбранного шаблона.

Область применения

config + inline Применима и в layouts.yml, и в команде.
inline only Применима только в команде; в конфигурации отклоняется, поскольку там не имела бы связного смысла.

Каждая аннотация требует узел прав minutiae.annotation.<name>; колонка узла ниже опущена, поскольку от этой конвенции нет отклонений. Аннотации из шаблона тоже проверяются по правам выдающего, так что шаблон не может протащить флаг мимо грантов младшего модератора.

Поведенческие флаги

Меняют то, что санкция делает с игроком в сессии. Все — config + inline.

@shadow Скрытая изоляция: игрок спрятан от остальных, его взаимодействия аннулируются, а чат отображается только ему самому. Бан с shadow впускает игрока и изолирует при входе, вместо отказа на пороге. Влечёт @silent.
@ghost Прячет игрока от других и аннулирует его взаимодействия с миром. В отличие от shadow, игроку ничего не скрывают о происходящем.
@rubberband Откатывает любое перемещение, превышающее настроенную длину привязи за ход.
@ip-lock Помечает санкцию для принуждения на уровне адреса по захваченным сигналам подключения.
@silent Подавляет публичное объявление санкции.
@transcript Захватывает и сохраняет недавние строки чата субъекта вместе с санкцией. Просматривается как CSV в веб-панели.

Модификаторы политики

Меняют то, как санкция вычисляется или стареет. Все — config + inline.

@escalate Включает лестницу эскалации шаблона против прецедента игрока по указанному правилу.
@warn-first(n) Первые n нарушений по правилу становятся формальными предупреждениями вместо меры. @now обходит ворота.
@decay(30d) Санкция перестаёт учитываться в прецеденте после истечения окна.
@probation(2w) Открывает окно пробации; рецидив внутри него отягчается согласно политике эскалации в config.yml.
@tariff(1d) Нижний порог срока, применяемый после эскалации. Только для временных мер.
@appeal(on|off) Может ли санкция быть обжалована. По умолчанию on.
@evidence Требование или приложение доказательств. Формы: голая, @evidence(required) или @evidence(url=..., type=...).
@notify(канал...) Направляет сводку санкции в названные каналы уведомлений.
@suspended(30d) Записывает условную санкцию: неактивна, пока другая санкция по тому же правилу внутри окна не активирует её. Несовместима с @stay.
@expunge(90d) Планирует удаление из видимости дела.
@review(7d) Планирует обязательный пересмотр персоналом; подошедшие пересмотры забираются один раз и уходят в каналы по умолчанию.

Директивы команды

Токены только для команды, потребляемые резолвером.

@measure(MUTE) Задаёт меру ручной санкции. Взаимоисключима с шаблоном, который определяет собственную.
@commute(SUSPENSION) Заменяет меру шаблона. Смягчающее и отягчающее направления несут раздельные узлы прав.
@count(P.4.1) Ссылается на дополнительное правило. Повторяема; ручная санкция повышает первый count до основного правила.
@remand Самостоятельная мера пресечения: карантин до принятия решения. Не сочетается ни с одним источником меры.
@reason(текст) Задаёт причину; эквивалент переопределения reason=.
@stay(2h) Откладывает активацию на окно; следующая санкция против игрока активирует ожидающие отсрочки. Несовместима с @suspended.
@now Обходит ворота warn-first: мера применяется сразу, независимо от квоты предупреждений.
@dry-run Разрешает и показывает полный исход, ничего не сохраняя и не применяя.
@as(имя) Атрибутирует санкцию другому имени персонала. Уровень админа.
@backdate(2d) Сдвигает время выдачи в прошлое. По матрице связей по умолчанию требует @evidence.
@link(ссылка) Прикрепляет внешнюю ссылку, например URL тикета.
@note(текст) Прикрепляет внутреннюю заметку по делу, никогда не показываемую субъекту.
@waive(имя) Снимает требование компаньона из матрицы связей для этой одной команды.
@amend(id) Изменяет существующую санкцию (правило, причину, срок, пункты) вместо выдачи новой.
@weave(#id) Присоединяет новую санкцию под существующее дело; участники цепочки разделяют прецедент.

Матрица связей

Аннотации связаны друг с другом тремя редактируемыми оператором отношениями в config.yml:

annotations:
  implies:
    shadow: [silent]        # применение @shadow применяет и @silent
  requires:
    backdate: [evidence]    # @backdate требует @evidence (снимаемо)
  conflicts: []             # неупорядоченные пары, несовместимые вместе
implies Транзитивное расширение: влекомое множество добавляется автоматически и показывается как (неявно) в отчётах.
requires Жёсткое требование компаньона, проверяемое после расширения. @waive(имя) снимает его на одну команду для держателей minutiae.waive.
conflicts Симметричное исключение; команда отклоняется с указанием обоих нарушителей.
Внимание

Цепочка импликаций, порождающая внутренний конфликт, сообщается при запуске, но не удаляется; отклонение происходит, когда команда фактически собирает конфликтующее множество. Следите за логом запуска после правки этой секции.

Аннотации при снятии

Команда снятия принимает замкнутое подмножество плюс одну директиву только для снятия: @silent, @dry-run, @note, @notify, @probation и @cascade. Семантика разобрана на странице Снятие санкций.
