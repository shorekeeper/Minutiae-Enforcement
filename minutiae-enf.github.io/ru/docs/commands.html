MEMinutiae Enforcement docs

Команды

Справочник грамматики выдачи санкций, семейства снятия, апелляций и диагностики. Узлы прав перечислены по формам; бандлы — на странице Права.

Выдача санкций

/enforce <игрок> [::шаблон] [@аннотация...] [ключ=значение...]

Требует minutiae.command.enforce. Спецификация — поток токенов в свободном порядке; каждый токен относится к одному из четырёх видов:

::key Выбирает настроенный шаблон, который поставляет правило, меру, срок, лестницу эскалации и аннотации по умолчанию. Не более одного на команду.
@name(...) Аннотация: поведенческий флаг или директива. Параметры — в скобках; !@name убирает аннотацию, пришедшую из шаблона.
key=value Скалярное переопределение. Распознаются duration и reason. Значения с пробелами берите в кавычки: reason="два слова".
всё остальное Игрок-цель. Требуется ровно один.
Примечание

Санкции нужна мера ровно из одного источника: шаблон, @measure(...) или @remand. Мера шаблона меняется через @commute(...), никогда через @measure.

Примеры

# Шаблон с его правилом, сроком и эскалацией:
/enforce Steve ::griefing

# Ручная санкция; временные меры требуют срок:
/enforce Steve @measure(MUTE) duration=3h reason="Флуд в чате"

# Смягчить меру шаблона, добавив пункт правил:
/enforce Steve ::cheating @commute(SUSPENSION) duration=30d @count(P.4.1)

# Просмотр без какого-либо применения:
/enforce Steve ::toxic @dry-run

Снятие санкций

/enforce lift <id> [vacate|pardon|served] [@аннотация...] [причина...]

Требует minutiae.command.lift. Вид решает, что снятая запись продолжает означать; полная модель — на странице Снятие санкций.

vacate Санкция выдана ошибочно. Запись полностью выходит из прецедента. Значение по умолчанию, когда вид не указан.
pardon Выдана правомерно, отменена как снисхождение. Прецедент сохраняется. Требует minutiae.lift.pardon.
served Досрочно завершена как отбытая. Прецедент сохраняется. Требует minutiae.lift.served.

Простые токены после вида образуют причину. Допустимые аннотации:

@silentПодавить публичное объявление о снятии.
@dry-runПросмотр, включая объём каскада.
@note(текст)Прикрепить внутреннюю заметку по делу.
@notify(канал...)Направить уведомление о снятии в каналы.
@probation(1w) Открыть окно пробации на снятой записи. Несовместимо с vacate, чья запись выходит из прецедента.
@cascade Распространить снятие на всю цепочку дел. Требует minutiae.lift.cascade.
Права

Две проверки зависят от цели: снятие санкции другого модератора требует minutiae.lift.foreign, а снятие бана — minutiae.lift.blocking. Обе действуют на каждом пути входа, включая принятие апелляции.

Откат снятия

/enforce unlift <id> [причина...]

Требует minutiae.lift.reinstate. Восстанавливает снятую санкцию: неистёкший бан удаляет игрока с сервера, а поведенческие ограничения применяются сразу.

Массовое снятие

/enforce liftall rule <id> [confirm [причина...]]
/enforce liftall staff <имя> [confirm [причина...]]

Требует minutiae.lift.bulk. Без confirm команда лишь сообщает, сколько санкций подпадает. Массовые снятия всегда являются аннулированием.

Просмотр

/enforce info <id> Полный вид одной санкции: статус, вид снятия, оставшийся срок, выдавший.
/enforce docket <игрок> [страница] Полная история санкций игрока.
/enforce appeals [страница] Апелляции, ожидающие разбора.
/enforce appeal <id> accept|deny [причина] Решить апелляцию. Принятие аннулирует санкцию.

Апелляции игроков

/appeal <id-санкции> <текст>

Доступна каждому игроку по умолчанию. Санкцию можно обжаловать один раз, пока апелляция в ожидании; идентификаторы автодополняются из недавних санкций самого игрока.

Диагностика

/me-ban открывает состояние подсистем: ping (проба хранилища), rules, layouts, annotations, fingerprint <игрок>, verbose on|off и reload. Каждая подкоманда несёт собственный узел под minutiae.command.*.
